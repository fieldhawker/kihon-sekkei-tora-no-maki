## 外部IF詳細

### IF-001: 決済サービス連携（Stripe）

**目的**: 商品購入時にクレジットカード決済を行う

**連携方式**: REST API（HTTPS）

| 項目 | 内容 |
|-----|-----|
| ベースURL | `https://api.stripe.com/v1` |
| 認証方式 | API Key（Secret Key を Authorization ヘッダーに設定） |
| タイムアウト | 30秒 |
| リトライ | 指数バックオフで最大3回 |
| エラー時の動作 | 決済失敗として注文ステータスを `failed` に更新する |

**主な連携エンドポイント**

| 操作 | エンドポイント | 送信データ | 受信データ |
|-----|------------|---------|---------|
| 支払いインテント作成 | `POST /payment_intents` | 金額・通貨 | payment_intent_id |
| 支払い確定 | `POST /payment_intents/:id/confirm` | payment_method_id | ステータス |

**エラーコードと対応**

| Stripeエラーコード | 意味 | システムの対応 |
|----------------|-----|------------|
| `card_declined` | カード決済拒否 | ユーザーに「カードが利用できません」と表示 |
| `insufficient_funds` | 残高不足 | ユーザーに「残高が不足しています」と表示 |
| `expired_card` | カード期限切れ | ユーザーに「カードの有効期限が切れています」と表示 |

### IF-002: メール配信サービス連携（SendGrid）

**目的**: ユーザー登録・注文確定時の通知メールを送信する

**連携方式**: REST API（HTTPS）

| 項目 | 内容 |
|-----|-----|
| ベースURL | `https://api.sendgrid.com/v3` |
| 認証方式 | API Key（Authorization ヘッダー） |
| タイムアウト | 10秒 |
| エラー時の動作 | メール送信失敗をログに記録し、処理は続行する（メール失敗で注文をキャンセルしない） |

**送信するメールの種類**

| テンプレートID | 送信タイミング | 送信先 | 件名 |
|------------|------------|------|-----|
| TMPL-001 | ユーザー登録完了時 | 登録ユーザー | 【システム名】ご登録ありがとうございます |
| TMPL-002 | 注文確定時 | 購入ユーザー | 【システム名】ご注文を承りました |
| TMPL-003 | パスワードリセット申請時 | 申請ユーザー | 【システム名】パスワードリセットのご案内 |

---

### IF-004: 売上集計システム連携（CSVファイル）

**目的**: 毎日の売上データを集計システムに連携する

**連携方式**: SFTP経由でのCSVファイル転送

| 項目 | 内容 |
|-----|-----|
| 転送先 | `sftp://集計システムサーバー/import/sales/` |
| 認証方式 | SSH公開鍵認証 |
| 実行タイミング | 毎日 0:30（バッチ B-001 の30分後） |
| ファイル名形式 | `sales_YYYYMMDD.csv` |
| 文字コード | UTF-8（BOM付き ← Excel対応のため） |
| 改行コード | CRLF |

**CSVフォーマット**

```
注文ID,注文日,ユーザーID,商品ID,商品名,数量,単価,合計金額
1001,2024-01-15,501,201,テスト商品A,2,1000,2000
1002,2024-01-15,502,202,テスト商品B,1,5000,5000
```
