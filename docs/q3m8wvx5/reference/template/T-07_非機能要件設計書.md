| 項目 | 要件値 | 測定方法 | 備考 |

## 2. 可用性要件

| 項目 | 要件値 | 実現方法 |
|-----|------|--------|
| 年間稼働率 | **99.9%以上**（月間ダウンタイム約43分以内） | マルチAZ構成・ヘルスチェック自動復旧 |
| 計画停止 | 月1回・深夜2:00〜4:00・2時間以内 | メンテナンス画面の表示 |
| 障害復旧目標時間（RTO） | **4時間以内** | オートスケーリング・フェイルオーバー設定 |
| 許容データ損失時間（RPO） | **1時間以内** | DBスナップショット 1時間ごと |

## 4. セキュリティ要件

| 要件 | 実現方法 |
|-----|--------|
| 通信の暗号化 | 全通信をHTTPS（TLS 1.2以上）で行う |
| 認証 | JWT（有効期限24時間）+ リフレッシュトークン |
| パスワード保存 | bcrypt（コスト係数12）でハッシュ化 |
| 個人情報の取り扱い | 個人情報保護法に準拠。ログへの個人情報出力禁止 |
| アクセス制御 | ロールベースアクセス制御（RBAC）を全APIに適用 |
| 脆弱性対策 | OWASP Top 10 の全項目への対策を設計書に明記 |
| ログイン試行制限 | 5回連続失敗で30分ロック |

## 5. 保守性・運用性要件

| 要件 | 実現方法 |
|-----|--------|
| ログ保存期間 | アプリケーションログ: 90日、アクセスログ: 1年 |
| ログ出力レベル | 本番: INFO以上、ステージング: DEBUG以上 |
| 監視 | CloudWatch / Datadog 等でCPU・メモリ・エラーレートを監視 |
| アラート | エラーレート 1%超または CPU 80%超で Slack 通知 |
| バックアップ | DB: 日次フルバックアップ + 1時間ごとスナップショット |
| バックアップ保存期間 | 30日間 |
| 障害対応SLA | P1（サービス停止）: 30分以内に初動対応 |
