Claude Code 活用方法50選¶

Claude Code（ターミナル・IDE統合型AIエージェント）を最大限活用するためのユースケース集。

コーディング支援¶

1. バグの原因特定と修正¶

活用例: エラーメッセージとスタックトレースをそのままターミナルに貼り付けて claude "このエラーを直して" と実行。Claude Codeがコードを自律的に読み込み、原因を特定して修正まで行う。

2. 新機能の実装依頼（自然言語で）¶

活用例: claude "ユーザー一覧ページにCSVエクスポートボタンを追加して" と依頼するだけで、関連ファイルを探索・編集・必要であれば新ファイル作成まで自律実行。

3. コードリファクタリング¶

活用例: claude "src/utils/以下の重複コードを整理して、関数を共通化して" と指示。複数ファイルをまたいだリファクタリングを一括で実施。

4. コードレビューと改善提案¶

活用例: claude "このPRの変更をレビューして、問題点と改善案を教えて" と依頼。セキュリティ・パフォーマンス・可読性の観点から指摘を得られる。

5. テストコードの自動生成¶

活用例: claude "src/services/user.ts のユニットテストをvitestで書いて" と指示。実装を読み込んだ上で、エッジケースを含むテストを自動生成。

6. 型定義の追加・修正（TypeScript）¶

活用例: claude "any型になっている箇所をすべて適切な型に直して" と指示。コードを横断して型の不整合を発見し修正。

7. APIエンドポイントの一括実装¶

活用例: 「OpenAPIのYAMLファイルを渡して claude "この仕様書を元にExpressのルーティングを実装して" と依頼。スキーマから実装まで自動生成。

8. データベースマイグレーションの作成¶

活用例: claude "usersテーブルにprofile_imageカラム（nullable text）を追加するマイグレーションを作って" と指示。Prisma/Drizzle/SQLの構文を自動で選択して生成。

9. 依存パッケージのアップグレード対応¶

活用例: claude "React 18から19へのアップグレードに伴う破壊的変更を調べて、対応が必要な箇所を修正して" と依頼。コードベース全体を横断して修正。

10. コードの動作説明（コードリーディング支援）¶

活用例: 読み解けないOSSのコードに対して claude "この認証ミドルウェアが何をしているか、処理の流れを順番に説明して" と依頼。

プロジェクト管理・設計¶

11. プロジェクト構造の把握と説明¶

活用例: 初めて触るリポジトリで claude "このプロジェクトのアーキテクチャと主要な処理の流れを説明して" と実行。ディレクトリ構造・依存関係・エントリーポイントを整理して説明。

12. CLAUDE.md（プロジェクト設定）の作成¶

活用例: claude を新規プロジェクトで実行した際に /init コマンドを使うと、コードベースを解析した上でプロジェクト固有のCLAUDE.mdを自動生成。チームの慣習・禁止事項・よく使うコマンドを記録。

13. タスクのToDoリスト管理（TodoWrite）¶

活用例: 複雑な実装タスクで claude "認証機能を実装して。工程をTodoに分解して進めて" と指示。Claude Codeが自律的にタスクを分解・逐次実行・完了チェックを行う。

14. 実装計画のレビューと修正¶

活用例: 設計書のドラフトをAIに読み込ませて claude "この設計のリスクと代替案を教えて" と依頼。着手前にフィードバックを得られる。

15. ドキュメントとコードの整合性チェック¶

活用例: claude "README.mdに書かれているAPIの仕様と実際のコードにズレがないか確認して" と依頼。乖離箇所を一覧で報告。

Git・バージョン管理¶

16. コミットメッセージの自動生成¶

活用例: git diff の内容をもとに claude "このdiffに対してConventional Commits形式のコミットメッセージを提案して" と依頼。変更の意図を的確に表すメッセージが生成される。

17. PRの説明文（本文）の自動生成¶

活用例: claude "このブランチの変更内容からGitHub PR用の説明文をMarkdown形式で書いて" と依頼。変更概要・テスト方法・スクリーンショット指示まで含めた本文が生成。

18. コンフリクトの解消¶

活用例: マージコンフリクトが発生したファイルに対して claude "このコンフリクトを解消して。基本はmainブランチ優先だが、機能追加部分は両方残して" と指示。

19. ブランチ戦略の相談¶

活用例: claude "このプロジェクトのリリースフローに合ったGitブランチ戦略を提案して" と依頼。チーム規模・デプロイ頻度を伝えると適切な戦略（GitHub Flow / GitFlow等）を推薦。

20. git bisect を使ったバグ特定の補助¶

活用例: claude "どのコミットでこのバグが混入したか特定するgit bisectの手順を教えて、実行も手伝って" と依頼。二分探索の手順をガイドしながら実行。

テスト・品質管理¶

21. E2Eテストシナリオの設計と実装¶

活用例: claude "ログイン〜商品購入〜注文確認までのE2EテストをPlaywrightで書いて" と依頼。ページ操作・アサーション・エラーケースを含む完全なテストを生成。

22. テストカバレッジの改善¶

活用例: claude "現在カバレッジが低いファイルを調べて、優先的にテストを追加して" と指示。カバレッジレポートを解析し、未テスト箇所に集中してテストを追加。

23. セキュリティ脆弱性のスキャンと修正¶

活用例: claude "このコードにSQLインジェクションやXSSの脆弱性がないか確認して、あれば修正して" と依頼。OWASP Top 10の観点で診断・修正。

24. パフォーマンスのボトルネック特定¶

活用例: claude "このAPIのレスポンスが遅い。コードを見てN+1問題や不要な処理がないか分析して" と依頼。問題箇所を特定して最適化案を提示。

25. リンター・フォーマッターの設定最適化¶

活用例: claude "このプロジェクトにESLint + Prettierを導入して。TypeScriptとReactの推奨設定で" と指示。設定ファイル生成からpackage.jsonのscripts追加まで自動実行。

自動化・スクリプト作成¶

26. シェルスクリプトの作成¶

活用例: claude "毎朝9時にS3バケットのログを圧縮してアーカイブフォルダに移すbashスクリプトを書いて" と依頼。エラーハンドリング・ログ出力付きのスクリプトを生成。

27. CI/CDパイプラインの構築¶

活用例: claude "GitHub ActionsでPR時に自動テスト・型チェック・Lintを走らせるワークフローを作って" と指示。YAMLファイルを生成し .github/workflows/ に配置。

28. データ変換スクリプトの作成¶

活用例: claude "このCSVを読み込んで、特定の列を変換・集計してJSON形式で出力するPythonスクリプトを書いて" と依頼。データの仕様を渡すだけで変換ロジックを生成。

29. 定期バッチ処理の実装¶

活用例: claude "毎日深夜に未処理の注文データを集計してSlackに通知するバッチを実装して" と依頼。スケジューラ設定・API連携・エラー通知まで一括実装。

30. Dockerfileの作成・最適化¶

活用例: claude "このNode.jsアプリのDockerfileをマルチステージビルドで最適化して。イメージサイズを最小化して" と指示。キャッシュ効率・セキュリティも考慮した設定を生成。

API・外部連携¶

31. REST API クライアントの自動生成¶

活用例: OpenAPIのYAMLを渡して claude "このAPIのTypeScript型付きクライアントを生成して" と依頼。型定義からリクエスト/レスポンスのラッパー関数まで自動生成。

32. WebhookのハンドラーとバリデーションIFの実装¶

活用例: claude "StripeのWebhookを受け取るエンドポイントを実装して。署名検証とイベント種別ごとの処理分岐も含めて" と指示。

33. 外部APIのエラーハンドリング強化¶

活用例: claude "このAPI呼び出し箇所にリトライ・タイムアウト・レートリミット対応を追加して" と依頼。堅牢なAPI連携コードに改善。

34. GraphQLスキーマとリゾルバの実装¶

活用例: claude "このRESTエンドポイントをGraphQL化して。型定義・リゾルバ・N+1対策（DataLoader）まで実装して" と一括依頼。

35. MCP（Model Context Protocol）サーバーの作成¶

活用例: claude "社内の〇〇APIをClaude Codeから呼べるようにするMCPサーバーを作って" と依頼。Claude Code自身がMCPサーバーのコードを生成し、外部ツール統合を実現。

データベース・インフラ¶

36. SQLクエリの最適化¶

活用例: 実行計画（EXPLAIN ANALYZE）の結果を貼り付けて claude "このクエリを最適化して。インデックスの追加提案も含めて" と依頼。

37. スキーマ設計のレビューと改善¶

活用例: ER図またはDDLを渡して claude "このスキーマ設計の正規化・インデックス・制約の問題点を指摘して" と依頼。

38. Terraformコードの作成¶

活用例: claude "AWSでECS + RDS + ALBの基本構成をTerraformで書いて。環境変数は変数化して" と指示。インフラ構成のコード化を自動化。

39. ログ解析スクリプトの作成¶

活用例: アクセスログのサンプルを渡して claude "エラーレートが高いパスと時間帯を集計するスクリプトを作って" と依頼。

40. バックアップ・リストア手順の自動化¶

活用例: claude "PostgreSQLのバックアップをS3に自動アップロードするスクリプトと、リストア手順書を作って" と依頼。

ドキュメント・ナレッジ管理¶

41. コードベースのドキュメント自動生成¶

活用例: claude "src/以下の主要なモジュールのJSDocコメントを追加して。外部公開APIになる関数を優先して" と指示。コメントを読み込んだ上で適切なドキュメントを生成。

42. ADR（Architecture Decision Record）の作成¶

活用例: claude "今回の技術選定（〇〇を採用した理由）をADR形式でドキュメント化して" と依頼。決定背景・代替案・トレードオフを整理したADRが生成される。

43. オンボーディングドキュメントの作成¶
