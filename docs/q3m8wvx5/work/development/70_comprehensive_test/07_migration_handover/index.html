STEP 7: 移行・引き継ぎ準備の確認¶

このSTEPの目的¶

本番リリースに向けた「移行手順」「引き継ぎ準備」が実際に機能することを確認すること。

このSTEPで確認するのは「手順書が存在するか」ではなく、「手順書通りに実際に動かして、問題なく機能するか」 です。
リリース当日に初めて手順を実行するのでは遅い。このSTEPでリハーサルとして確認します。

確認項目一覧¶

# 確認項目 担当者 完了確認
1 データ移行手順の確認 □
2 移行後のデータ整合性確認 □
3 切り替え手順の確認 □
4 ロールバック手順の確認 □
5 業務担当者への引き継ぎ確認 □
6 運用・保守手順書の確認 □

1. データ移行手順の確認¶

内容¶

旧システムから新システムへのデータ移行手順を、テスト環境で実際に実行して確認します。

確認項目¶

確認項目 確認内容 結果
移行手順書の存在 データ移行手順書が作成されているか
手順書通りに実行できるか 手順書の手順を1ステップずつ実行し、問題なく進められるか
移行ツールの動作 移行ツールがエラーなく完了するか
移行時間 本番データ量での移行にかかる時間が許容範囲か

移行時間の記録¶

データ種類 データ件数 移行開始時刻 移行完了時刻 移行時間 許容範囲 結果
顧客マスタ 1,000件 10:00 10:05 5分 30分以内 OK
受注データ 50,000件 10:10 11:00 50分 2時間以内 OK

2. 移行後のデータ整合性確認¶

内容¶

移行後のデータが、旧システムのデータと一致していることを確認します。

確認手順¶

  1. 旧システムのデータ件数を記録する
  2. 移行ツールを実行する
  3. 新システムのデータ件数を確認し、旧システムと一致するか確認する
  4. 抜き取りでデータの内容を確認する（全件確認が難しい場合は代表的なデータを確認）

確認記録シート¶

データ種類 旧システム件数 新システム件数 一致 抜き取り確認
顧客マスタ 1,000件 1,000件 ○ 確認済み（10件抜き取り：全件一致）
商品マスタ 500件 500件 ○ 確認済み（5件抜き取り：全件一致）
受注データ 50,000件 49,995件 × 5件欠損あり→BUG-010として報告

3. 切り替え手順の確認¶

内容¶

旧システムから新システムへの切り替え手順が明確で、問題なく実行できることを確認します。

切り替え手順書で確認すること¶

確認項目 確認内容
手順の明確さ 誰でも手順通りに実行できる粒度で書かれているか
切り替えのタイミング いつ・どの順番で切り替えるかが明確か
各ステップの担当者 誰が何を実施するかが明確か
確認ポイント 各ステップ後に「正しく完了したか」を確認する方法が記載されているか

切り替え手順の例と確認ポイント¶

【本番切り替え手順】（例）

1. 旧システムを停止する
   → 担当: インフラ担当　確認: 旧システムにアクセスできないことを確認

2. 新システムに旧システムのデータを移行する
   → 担当: 移行担当　確認: 移行ツールが正常終了し、件数が一致することを確認

3. 新システムのDNS設定を切り替える
   → 担当: インフラ担当　確認: 新システムのURLにアクセスできることを確認

4. 動作確認テストを実施する
   → 担当: テスト担当　確認: 主要な業務フローが正常に動作することを確認

5. 業務担当者に切り替え完了を連絡する
   → 担当: PL　確認: 業務担当者が新システムを使い始めたことを確認

4. ロールバック手順の確認¶

ロールバックとは¶

本番リリース後に重大な問題が発生したとき、旧システムへ戻す（元の状態に戻す）ための手順です。

なぜ必要か?
万が一、本番リリース後に「業務が止まる問題」が発見された場合、旧システムへ戻すことで被害を最小限に抑えられます。「ロールバックできない」状態でリリースすることは非常にリスクが高い。

ロールバック手順書で確認すること¶

確認項目 確認内容
手順の存在 ロールバック手順書が作成されているか
実行可否の確認 実際にロールバック手順を実行し、旧システムの状態に戻せるか（テスト環境で確認）
ロールバック時間 ロールバックにかかる時間が業務許容範囲か
データの扱い 新システム稼働後に更新されたデータはどう扱うか

ロールバック手順確認記録¶

確認項目 結果
ロールバック手順書の存在 あり（バージョン: v1.0）
テスト環境でのロールバック実施 実施済み（2024/03/20 14:00）
ロールバックにかかった時間 約15分（許容範囲: 1時間以内）
旧システムでの動作確認 OK
ロールバック後のデータ整合性 確認済み（欠損なし）

5. 業務担当者への引き継ぎ確認¶

確認項目¶

確認項目 確認内容 結果
操作マニュアルの完成 最新の操作マニュアルが完成しているか
研修の実施 業務担当者への研修・説明会が実施されたか
研修後の理解度確認 業務担当者がシステムを使えるか確認できたか
問い合わせ先の案内 困ったときの連絡先（ヘルプデスク・担当者）が案内されているか
よくある質問・対処方法 業務担当者がよく直面するであろう問題と対処方法がまとめられているか

6. 運用・保守手順書の確認¶

確認項目¶

確認項目 確認内容 結果
障害対応手順書 障害が発生した場合の対応手順が記載されているか
バッチ監視手順書 バッチ処理の実行確認・エラー対応手順が記載されているか
定期メンテナンス手順書 DB保守・バックアップ等の定期作業手順が記載されているか
問い合わせ対応フロー ユーザーからの問い合わせを誰がどう対応するかが明確か
連絡体制 緊急時の連絡先・エスカレーションフローが明確か

次のSTEPへ進む前のチェックリスト¶

  • データ移行手順を実際に実行し、正常に完了することを確認した
  • 移行後のデータ整合性を確認した（件数・内容）
  • 切り替え手順書を確認し、手順通りに実行できることを確認した
  • ロールバック手順を実際に実行し、正常に戻せることを確認した
  • 業務担当者への研修・引き継ぎが完了していることを確認した
  • 運用・保守手順書が揃っていることを確認した
  • 確認結果をリーダーに報告した

上記がすべてチェックできたら → STEP 8: テスト完了確認・成果物の提出 へ進む
