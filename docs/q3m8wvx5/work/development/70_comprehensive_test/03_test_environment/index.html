STEP 3: テスト環境・テストデータの準備¶

このSTEPの目的¶

本番に近い環境とデータを整え、運用テストを正しく実施できる状態にすること。

環境やデータが不十分なまま運用テストを実施しても、「本番では問題が出ないか」を確認できません。
このSTEPは時間がかかることが多いため、STEP 2（シナリオ作成）と並行して早めに着手してください。

準備するもの 全体チェックリスト¶

# 準備項目 担当者 完了予定日 完了確認
1 テスト環境の構成確認 □
2 テストデータの準備 □
3 操作マニュアル・手順書の確認 □
4 監視・ログ収集の確認 □

1. テスト環境の確認¶

確認項目¶

確認項目 確認内容 確認結果
環境の独立性 本番環境・開発環境・結合テスト環境と混同していないか
サーバー構成 本番と同等のサーバー構成（台数・スペック）か
データベース 本番と同等のDB種類・バージョンか
ネットワーク 本番と同等のネットワーク構成か
外部連携先 外部システムとの連携がテスト環境で動作するか（またはモック化されているか）
アプリケーションバージョン 最新のリリース候補バージョンがデプロイされているか

環境が混同しないようにする¶

環境名 用途 注意
本番環境 実際のサービス稼働 絶対にテストで使わない
ステージング環境（運用テスト環境） 本番に最も近い環境。今回の運用テストで使う
結合テスト環境 結合テストで使った環境。混同しない
開発環境 開発者が動作確認する環境

ポイント: 接続先URLやDB名を必ず確認し、「今自分が操作しているのがどの環境か」を明確にしてから作業を始める。

2. テストデータの準備¶

テストデータの種類¶

種類 説明 例
正常データ 正常系シナリオで使うデータ 在庫10件の商品データ、有効な顧客データ
境界値データ 最大値・最小値・ゼロなど境界付近のデータ 在庫が0件・1件・最大値の商品データ
異常データ 異常系シナリオで使うデータ 存在しない顧客コード、在庫不足の商品
大量データ 性能テストで使う本番相当のデータ量 受注データ1万件
移行データ 旧システムから移行するデータ 旧システムの顧客データ100件

テストデータ準備時の注意点¶

個人情報・機密情報の取り扱い: - 本番データをそのまま使ってはいけません - 個人情報（氏名・住所・電話番号）はマスキング（ダミーデータに置き換え）する - マスキングの方法はリーダーに確認する

マスキングの例:

本番データ マスキング後
山田 太郎 テスト 太郎
東京都渋谷区XXX テスト都テスト市XXX
090-1234-5678 090-0000-0000
test@example.com test001@test.example.com

テストデータ準備チェックリスト¶

  • 正常系シナリオ用のデータが準備できている
  • 異常系シナリオ用のデータが準備できている（在庫不足・存在しないデータなど）
  • 性能テスト用の大量データが準備できている（テスト計画書の要件に従った件数）
  • 個人情報・機密情報はマスキング済み
  • 移行テスト用の旧システムデータが準備できている（該当する場合）
  • テストデータの初期化・リセット手順が決まっている

3. 操作マニュアル・手順書の確認¶

確認項目¶

確認項目 確認内容 確認結果
最新版の確認 現在のシステムの仕様に合った最新のマニュアルか
操作手順の正確性 マニュアルの手順通りに操作して、問題なく進められるか
用語の統一 システム上の名称とマニュアルの名称が一致しているか
対象読者への適切さ 業務担当者が読んで理解できる内容か

マニュアルに不備があった場合¶

マニュアルに不備を見つけた場合は、この段階で修正する。
運用テスト中に発見した場合も、不具合票（STEP 5参照）に記録して修正する。

4. 監視・ログ収集の確認¶

確認項目¶

確認項目 確認内容 確認結果
アプリケーションログ テスト実施中のエラーや警告をログで確認できるか
パフォーマンスログ 処理時間・リソース使用率（CPU・メモリ）を確認できるか
バッチ実行ログ バッチ処理の実行結果・実行時刻がログに記録されるか
エラー通知 重大なエラーが発生した場合に通知が届くか

ログの確認方法¶

ログの確認方法・ログファイルの場所はリーダーまたは担当者に事前に確認しておく。
テスト中にエラーが発生したとき、すぐにログを確認できる状態にしておく。

環境準備 完了確認レポートの例¶

よくある問題と対処法¶

環境が本番と大きく異なる場合:
リーダーに報告し、差異が「テスト結果に影響するか」を判断してもらう。「環境が違うからテストの意味がない」と自己判断せず、必ず報告する。

テストデータの準備に時間がかかる場合:
大量データの生成・データ移行の準備は時間がかかることが多い。STEP 2（シナリオ作成）と並行して早めに着手する。データ生成ツールの使い方はリーダーに確認する。

マニュアルが存在しない場合:
「マニュアルが存在しないこと」自体が問題なので、リーダーに報告する。運用テスト実施前にマニュアルを作成してもらう必要がある。

次のSTEPへ進む前のチェックリスト¶

  • テスト環境の構成を確認し、本番相当の環境が整っている
  • 全シナリオで必要なテストデータが揃っている
  • 個人情報・機密情報のマスキングが完了している
  • 操作マニュアルが最新版であることを確認した
  • ログ確認方法・監視環境が整っている
  • 環境準備の完了をリーダーに報告した

上記がすべてチェックできたら → STEP 4: テストの実施 へ進む
