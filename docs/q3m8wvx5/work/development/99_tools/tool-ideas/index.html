1. 画面ワイヤーフレームビルダー¶

ファイル候補: wireframe-builder.html

GUIで画面部品（テキストボックス・ボタン・テーブル・モーダル等）をドラッグ＆ドロップして簡易ワイヤーフレームを作成するHTMLツール。

  • 主な機能:
  • 部品パレット（入力欄・ボタン・ラベル・テーブル・ナビゲーションバー等）
  • キャンバスへのドラッグ配置＋テキスト編集
  • PNG/SVGエクスポート、またはHTMLとしてコピー
  • 用途: STEP2「各画面のワイヤーフレームレベルのレイアウト作成」を素早く行う

2. APIテスト送信フォーム（APIモックテスター）¶

ファイル候補: api-tester.html

設計書のAPIエンドポイントに対してフォームUIからHTTPリクエストを送信し、レスポンスを確認するHTMLツール。

  • 主な機能:
  • エンドポイントURL・HTTPメソッド・ヘッダー（Authorizationトークン等）を入力
  • リクエストボディをフォーム形式またはJSONエディタで入力
  • レスポンス（ステータスコード・ヘッダー・ボディ）をビジュアル表示
  • リクエスト履歴の保存（localStorageベース）
  • 用途: STEP3「詳細API設計」時の動作確認・フロント/バック間の疎通確認

3. バリデーションチェックリスト生成ツール¶

ファイル候補: validation-checklist.html

入力項目のバリデーション条件を選択・入力すると、実施すべきテストケースを網羅的に自動生成するHTMLツール。

  • 主な機能:
  • 項目属性の選択（必須/任意・文字種・文字数・形式など）
  • 同値分割法・境界値分析に基づくテストケースの自動生成
  • 生成結果をMarkdown/CSV形式でコピー・ダウンロード
  • 入力条件の例:
  • 必須 / 任意
  • 文字種: 半角英数字のみ / 全角可 / 数値のみ / メール形式 / 日付形式 等
  • 文字数: 最小N文字・最大M文字
  • 数値範囲: N以上・M以下
  • 用途: STEP2「各画面の入力項目・バリデーションルール定義」後のテスト設計

4. 画面遷移図ビジュアライザー¶

ファイル候補: screen-flow-visualizer.html

画面一覧（画面ID・画面名・遷移先）をテーブル形式で入力すると、画面遷移図をSVGで自動描画するHTMLツール。

  • 主な機能:
  • 画面ノード＋矢印（遷移）のグラフ自動生成
  • ノードのドラッグ移動で図の整形
  • SVG/PNG形式でエクスポート
  • 用途: STEP2「画面遷移図（全体）の作成」を設計書に貼り付けられる形で出力

5. テーブル定義書プレビューツール¶

ファイル候補: table-definition-preview.html

Markdown形式のテーブル定義書をペーストすると、列定義・制約・インデックスを見やすくレンダリングするHTMLツール。

  • 主な機能:
  • Markdownテーブルのパース＆整形表示
  • NOT NULL / PK / FK / UNIQUE などのバッジ表示
  • テーブル間のER図（簡易）の生成
  • 用途: STEP3「テーブル定義書・ER図」レビュー時の視認性向上

6. OpenAPI YAMLビューア＆テスター¶

ファイル候補: openapi-viewer.html

OpenAPI（YAML/JSON）をペーストすると、SwaggerUI風にエンドポイント一覧を表示し、その場でリクエスト送信もできるHTMLツール。

  • 主な機能:
  • YAML/JSONパースとエンドポイント一覧表示
  • エンドポイントごとのリクエスト実行フォーム（SwaggerUI相当）
  • レスポンスのインライン表示
  • 用途: STEP3「OpenAPI仕様書の完全版作成」後の確認・チームレビュー

7. ADRテンプレート生成ツール¶

ファイル候補: adr-generator.html

フォームに必要事項（タイトル・背景・決定事項・代替案・結果）を入力すると、ADRのMarkdownを自動生成してコピーできるHTMLツール。

  • 主な機能:
  • ADR番号・タイトル・ステータス・日付の入力
  • 背景・決定事項・根拠・代替案・影響の各セクション入力
  • Markdown形式でプレビュー＆クリップボードコピー
  • 用途: アーキテクチャ決定（フレームワーク選定等）の記録

8. トレーサビリティマトリクスチェッカー¶

ファイル候補: traceability-checker.html

要件ID・機能ID・画面ID・API ID・テストID を対応付けて入力し、未紐付けや抜け漏れを可視化するHTMLツール。

  • 主な機能:
  • IDのインポート（CSV貼り付け）
  • マトリクス表示（行: 要件、列: 設計成果物）
  • 未対応の要件・テスト未網羅の機能をハイライト
  • 用途: STEP8「取りまとめレビュー」でのT-08「トレーサビリティマトリクス」確認

9. エラーコード管理ツール¶

ファイル候補: error-code-manager.html

エラーコード・HTTPステータス・メッセージ・発生箇所をフォームで登録・管理し、一覧表示とMarkdown出力を行うHTMLツール。

  • 主な機能:
  • エラーコードの追加・編集・削除（localStorageで保存）
  • HTTPステータス別のフィルタ表示
  • Markdown形式のエラーコード一覧としてエクスポート
  • 用途: STEP2「エラー処理方針・エラーコード体系の定義」の一元管理

10. 権限マトリクスエディタ¶

ファイル候補: permission-matrix-editor.html

ロール（ユーザー種別）と機能・画面・APIをマトリクス形式で管理し、○×△で権限を設定してMarkdown出力するHTMLツール。

  • 主な機能:
  • ロール行と機能列のスプレッドシート風エディタ
  • ○（許可）/ ×（禁止）/ △（条件付き）の切り替え
  • Markdown/CSVエクスポート
  • 用途: T-05「権限マトリクス」テンプレートの入力・管理

要件定義フェーズ向けツール（追加アイデア）¶

顧客との口頭レベルのやり取りを正式な要件に落とし込むためのツール群。

11. ヒアリングシート＆要件起こしアシスト¶

ファイル候補: requirement-extractor.html

顧客との打ち合わせメモや会話ログを貼り付けると、要件定義書に必要な構造（目的・対象ユーザー・機能要件・非機能要件・制約・未確認事項）に整理・分類する支援ツール。

  • 主な機能:
  • フリーテキスト（打ち合わせメモ）の入力欄
  • 「誰が」「何をしたい」「なぜ」「条件は」のキーワードをハイライト
  • 抽出した内容を要件カード（機能要件 / 非機能要件 / 制約 / 未確認事項）に振り分けるUIで整理
  • 整理結果をMarkdown形式（要件定義書の骨格）でエクスポート
  • 用途: ヒアリング後すぐに要件の骨格を作る。「言った言わない」防止のための構造化

12. ユーザーストーリー作成ツール¶

ファイル候補: user-story-builder.html

「〇〇として、〜したい。なぜなら…」のユーザーストーリー形式で要件を記述・管理するツール。口頭説明をユーザー視点の文章に変換しやすい。

  • 主な機能:
  • ロール（As a...）/ ウォント（I want to...）/ ベネフィット（So that...）の3フィールド入力
  • 受け入れ条件（Acceptance Criteria）をチェックリスト形式で追加
  • 優先度（MoSCoW: Must / Should / Could / Won't）と担当者の設定
  • ストーリーの一覧表示・並び替え・フィルタ
  • Markdown / CSV エクスポート
  • 用途: 要件をユーザー視点で整理し、開発チームと共通認識を作る

13. 未確認事項・宿題管理ツール（課題管理）¶

ファイル候補: open-issues-tracker.html

ヒアリング中に出てきた「あとで確認する」「顧客に聞き返す必要がある」事項を即座に記録・管理するツール。

  • 主な機能:
  • 未確認事項の素早い登録（タイトル・内容・確認先・期限）
  • ステータス管理（未対応 / 確認中 / 解決済）
  • 解決内容・決定事項の記録
  • ミーティング単位でのグルーピング
  • Markdown形式の議事録・課題一覧としてエクスポート
  • 用途: 打ち合わせ中にリアルタイムで課題を拾い、次回ヒアリングまでにクリアにする

14. 要件優先度マトリクス（MoSCoW × 影響度）¶

ファイル候補: requirement-priority-matrix.html

洗い出した要件をMoSCoW法（Must / Should / Could / Won't）と実装コスト・業務影響度で評価し、優先順位付けを視覚化するツール。

  • 主な機能:
  • 要件一覧の入力（ID・要件名・説明）
  • MoSCoWカテゴリの選択
  • 実装コスト（小/中/大）と業務影響度（低/中/高）のスコアリング
  • 2軸バブルチャート（コスト vs 影響度）でのマッピング表示
  • 優先度スコア順の一覧表示・Markdownエクスポート
  • 用途: 顧客と「何を先に作るか」の合意形成に使う

15. ヒアリングチェックリスト（要件確認漏れ防止）¶

ファイル候補: requirement-checklist.html

要件定義ヒアリングで確認すべき観点（機能・非機能・制約・運用・移行・権限など）をカテゴリ別にチェックリスト化し、聞き漏れを防ぐツール。

  • 主な機能:
  • カテゴリ別チェックリスト（業務フロー / 画面 / データ / 外部連携 / 非機能 / 運用 / セキュリティ 等）
  • 各項目に「確認済 / 未確認 / N/A」とメモ欄
  • 確認状況のプログレスバー表示
  • プロジェクト固有の質問を追加できるカスタム欄
  • Markdown形式の確認済みレポートとしてエクスポート
  • 用途: ヒアリング前の準備・ヒアリング中の進行・終了後の抜け漏れ確認

16. 要件スコープ境界線エディタ（スコープIN/OUT管理）¶

ファイル候補: scope-boundary-editor.html

要件・機能・画面を「スコープIN（今回作る）/ OUT（今回は作らない）/ 将来検討」に仕分けし、顧客との認識合わせに使うツール。

  • 主な機能:
  • 項目カードをIN / OUT / 将来検討の3ゾーンにドラッグ＆ドロップ
  • OUTの理由・将来検討の条件をメモとして記録
  • スコープ一覧のMarkdown / CSV エクスポート
  • 変更履歴の記録（いつ・誰が・何をINからOUTに変えたか）
  • 用途: 要件の膨張（スコープクリープ）を防ぐための顧客合意文書の土台

優先度の考え方¶

優先度 ツール 理由
高 バリデーションチェックリスト生成（#3） テスト設計の抜け漏れ防止に直結
高 APIテスト送信フォーム（#2） API設計確認に即日使える
高 画面ワイヤーフレームビルダー（#1） 画面設計の最初に必要
中 画面遷移図ビジュアライザー（#4） 遷移図作成の手間削減
中 権限マトリクスエディタ（#10） 権限設計は全プロジェクト共通
中 エラーコード管理ツール（#9） API設計と並行して使う
低 OpenAPI YAMLビューア（#6） Swagger公式で代替可能
低 ADR生成ツール（#7） テンプレートで代替可能
低 テーブル定義書プレビュー（#5） Markdown Previewで代替可能
低 トレーサビリティチェッカー（#8） スプレッドシートで代替可能
