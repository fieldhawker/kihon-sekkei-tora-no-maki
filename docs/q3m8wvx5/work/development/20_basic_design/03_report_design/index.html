帳票設計¶

このドキュメントの目的¶

帳票設計は、システムから印刷・出力されるドキュメント（帳票）のレイアウトと出力条件を定義する作業です。

帳票（ちょうひょう）とは？
注文書、請求書、納品書、売上集計レポートなど、業務で使用する文書のことです。
紙への印刷、PDFダウンロード、Excel出力などが代表的な出力形式です。

作業の流れ¶

1. 帳票一覧の作成
2. 各帳票のレイアウト設計
3. 出力条件・タイミングの定義
4. データソースの定義

STEP 1: 帳票一覧の作成¶

帳票の洗い出し方¶

要件定義書と業務フロー図を参照し、以下の観点で帳票を洗い出します。

  • 業務フローの各ステップで発行・使用される文書はあるか？
  • ユーザーへのヒアリング内容に「〜書」「〜表」「〜レポート」という言葉が出ていたか？
  • 現在（As-Is）の業務で使っている帳票は何か？

帳票IDの採番ルール（例）¶

RPT-[カテゴリ]-[連番3桁]
例:
  RPT-ORD-001: 注文確認書
  RPT-INV-001: 請求書
  RPT-RPT-001: 月次売上レポート

帳票一覧の記載例¶

帳票ID 帳票名 概要 出力形式 出力タイミング 備考
RPT-ORD-001 注文確認書 注文内容を顧客に通知する PDF 注文確定時 メール添付
RPT-INV-001 請求書 月次の請求内容を記載する PDF 毎月末日 郵送・PDF両対応
RPT-DLV-001 納品書 商品の納品を証明する PDF 発送時 紙印刷して同梱
RPT-RPT-001 月次売上レポート 月別の売上集計を表示する Excel 翌月1日 管理者のみ
RPT-RPT-002 在庫一覧表 現在の在庫数量を一覧表示する Excel/CSV 随時（手動）

STEP 2: 帳票レイアウトの設計¶

レイアウト設計の目的¶

帳票のどこに・何の情報を・どのような形式で表示するかを定義します。
開発者がこれを見て帳票生成プログラムを実装できる精度が必要です。

帳票の構成要素¶

帳票は一般的に以下の3つのエリアで構成されます。

┌────────────────────────────────────┐
│              ヘッダー部              │  ← 帳票タイトル、会社情報、帳票番号、日付など
├────────────────────────────────────┤
│              明細部                  │  ← 繰り返し出力される行データ（商品明細など）
├────────────────────────────────────┤
│              フッター部              │  ← 合計金額、注意事項、承認欄など
└────────────────────────────────────┘

帳票レイアウト定義書の記載例（請求書）¶

帳票ID: RPT-INV-001
帳票名: 請求書
用紙サイズ: A4縦
出力形式: PDF

│                                              │
│  株式会社〇〇〇〇                              │  ← 自社名（固定）
│  〒000-0000 東京都〇〇区〇〇 1-2-3            │  ← 自社住所（固定）
│                                              │
│               請  求  書                     │  ← タイトル（固定）
│                                              │
│  請求番号: INV-[請求番号]   発行日: [発行日]   │  ← 動的データ
│                                              │

├─────────────┬─────┬──────┬──────────────────┤

├─────────────┼─────┼──────┼──────────────────┤
│ [商品名]     │[数量]│[単価]│ [金額]          │  ← 明細行（繰り返し）

項目定義¶

エリア 項目名 データソース 表示形式 備考
ヘッダー 請求番号 invoices.invoice_no そのまま
ヘッダー 発行日 invoices.issue_date YYYY年MM月DD日
ヘッダー 得意先名 customers.name そのまま
明細 商品名 order_items.product_name そのまま 繰り返し
明細 数量 order_items.quantity 数値（カンマなし） 繰り返し
明細 単価 order_items.unit_price ¥X,XXX 繰り返し
明細 金額 数量 × 単価（計算） ¥X,XXX 繰り返し
フッター 小計 明細金額の合計（計算） ¥X,XXX
フッター 消費税 小計 × 0.1（計算） ¥X,XXX 端数は切り捨て
フッター 合計 小計 + 消費税（計算） ¥X,XXX
フッター 支払期限 invoices.payment_due_date YYYY年MM月DD日

STEP 3: 出力条件・タイミングの定義¶

出力条件の定義¶

帳票をいつ・誰が・どんな操作で出力するかを定義します。

定義例（請求書）¶

## 出力条件

- **出力タイミング:** 毎月末日の24:00に自動バッチ処理で生成する
- **出力トリガー:** バッチスケジューラーによる自動実行
- **対象データ:** 当月中に「確定」ステータスになった注文
- **出力先:** /output/invoices/YYYY-MM/ フォルダに保存
- **ファイル名:** INV-[請求番号]-YYYYMMDD.pdf

## 手動出力

- 管理画面の「請求書再発行」ボタンからも個別に出力可能
- 権限: 管理者ロールのみ

出力形式別の留意点¶

PDF出力¶

  • ページ区切りの制御（明細が多い場合の改ページ）
  • フォント埋め込みの有無（別PCで開いたときに文字化けしないか）
  • 印刷時の余白設定

Excel出力¶

  • シート構成（1シートか複数シートか）
  • セルの書式設定（数値、日付のフォーマット）
  • 集計関数の扱い（固定値として出力するか、数式を含めるか）

CSV出力¶

  • 文字コード（UTF-8 / Shift-JIS など。Excelで開くならShift-JISが無難）
  • BOMの有無（Excelで開く場合はBOMあり推奨）
  • ヘッダー行の有無
  • 区切り文字（カンマ、タブなど）

成果物チェックリスト¶

  • 業務フローに登場するすべての帳票が洗い出されているか
  • 全帳票に一意のIDが付与されているか
  • レイアウト図と項目定義が全帳票分作成されているか
  • データソース（どのテーブル・カラムから取得するか）が明示されているか
  • 計算項目の計算ロジックが記載されているか
  • 出力タイミング・トリガーが定義されているか
  • 出力形式（PDF/Excel/CSV）が定義されているか

よくある失敗¶

「現行踏襲」で深く考えない¶

既存の帳票をそのまま引き継ぐ場合でも、「この項目は本当に必要か」「この計算式は正しいか」を確認します。現行帳票に誤りが含まれている場合もあります。

データソースを未定義にする¶

「どこから取ってくるか後で決める」にすると、開発時に困ります。帳票設計の段階でデータ設計と照らし合わせ、取得元を明確にします。

ページ区切りを考慮しない¶

明細行が多いときに帳票が複数ページにまたがるケースを想定しておかないと、印刷時に途中で切れてしまいます。

成果物例: 社員管理システム¶

帳票一覧¶

帳票ID 帳票名 概要 出力形式 出力タイミング 備考
RPT-EMP-001 社員名簿 全社員または条件指定社員の一覧を出力する PDF / Excel 随時（手動） 人事担当者のみ
RPT-EMP-002 社員個人票 個別社員の詳細情報を1枚にまとめた帳票 PDF 随時（手動）
RPT-DEP-001 部署別人員一覧 部署ごとの在籍社員数・一覧を出力する Excel 随時（手動） 管理職以上

帳票レイアウト定義書（社員名簿）¶

帳票ID: RPT-EMP-001 帳票名: 社員名簿 用紙サイズ: A4横 出力形式: PDF / Excel

明細 入社日 employees.hire_date YYYY年MM月DD日 繰り返し
フッター 合計件数 出力対象レコード数（計算） XX 名

出力条件¶

## 出力条件
- 出力タイミング: 帳票出力画面（SCR-RPT-001）での手動操作
- 権限: 人事担当者ロール以上
- 対象データ: 画面の検索条件に合致する在籍中の社員
- デフォルト: 全部署・全雇用形態・在籍中のみ

## ソート順
- 部署コード昇順 → 社員番号昇順

## ページ区切り
- 1ページあたり40行。超過した場合は改ページし、次ページにヘッダー行を再出力する

## ファイル名
- PDF: EmpList_YYYYMMDD.pdf
- Excel: EmpList_YYYYMMDD.xlsx
