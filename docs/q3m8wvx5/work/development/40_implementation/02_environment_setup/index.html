STEP 2: 開発環境のセットアップ¶

このSTEPの目的¶

自分のPC上でコードを書いて動かせる環境を整えます。
環境が整っていないと、コードを書いても動作確認ができません。

開発環境とは¶

「開発環境」とは、コードを書いて動作確認するための環境のことです。
本番環境（実際にユーザーが使う環境）とは別の、自分専用の環境です。

【環境の種類】

ローカル環境（自分のPC）← ここを整えるのが STEP 2
    ↓ 動いたら
開発環境（チーム共有サーバー）
    ↓ テスト済みなら
ステージング環境（本番と同じ構成で最終確認）
    ↓ 問題なければ
本番環境（実際のユーザーが使う）

セットアップの手順¶

1. セットアップ手順書を確認する¶

プロジェクトには必ず環境構築の手順書があります。まずそれを読みます。

確認する場所: - プロジェクトのリポジトリにある README.md - チームのWiki・Confluence・Notionなど - 先輩エンジニアに直接聞く

注意: 手順書を読まずに「なんとなく」セットアップしようとすると、
必要なステップを飛ばしてエラーが出て、何が原因か分からなくなります。
必ず手順書を見ながら1ステップずつ進めてください。

2. リポジトリの取得（clone / pull）¶

プロジェクトのソースコードをGitから取得します。

初めて取得する場合（clone）:

# GitHubなどのリポジトリURLをコピーして実行
git clone https://github.com/your-org/your-project.git

# 取得したフォルダに移動
cd your-project

すでに取得済みで最新化したい場合（pull）:

# 最新のコードを取得
git pull origin main

ポイント: 毎日作業を始める前に git pull して最新状態にする習慣をつけましょう。

3. 依存パッケージのインストール¶

プロジェクトが使っているライブラリ（外部ツール）をインストールします。
どのコマンドを使うかはプロジェクトの技術スタックによって異なります。

言語・フレームワーク別のインストールコマンド例:

言語・環境 インストールコマンド 定義ファイル
JavaScript / Node.js npm install または yarn install package.json
Python pip install -r requirements.txt requirements.txt
Java / Gradle ./gradlew build build.gradle
Ruby bundle install Gemfile
PHP / Composer composer install composer.json

ポイント: コマンドを実行してエラーが出た場合、エラーメッセージをそのままコピーして
チームメンバーや先輩エンジニアに見せると、原因を特定してもらいやすくなります。

4. 環境変数・設定ファイルの配置¶

アプリケーションはデータベースの接続情報やAPIキーなど、
環境ごとに異なる設定値を外部ファイルで管理しています。

よく使われるファイル名: - .env - config/local.yml - application-local.properties

設定手順:

  1. テンプレートファイルをコピーする 

    # .env.example を .env にコピー（よく使われるパターン）
cp .env.example .env

  2. コピーしたファイルを開いて、自分の環境の値を設定する 

    # .env の例
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp_development
DB_USER=postgres
DB_PASSWORD=（自分のローカルのDBパスワード）

  3. 設定値は先輩や手順書から教えてもらう

重要: .env などの設定ファイルはGitにコミットしてはいけません。
データベースのパスワードやAPIキーなど機密情報が含まれるためです。
.gitignore に記載されているか確認しましょう。

5. データベースのセットアップ¶

アプリケーションがデータベースを使う場合、ローカルにもDBを用意します。

典型的な手順（プロジェクトによって異なります）:

# 1. データベースを作成する
rails db:create       # Ruby on Rails の場合
python manage.py migrate  # Django の場合

# 2. テーブルを作成する（マイグレーション）
rails db:migrate
python manage.py migrate

# 3. 初期データを投入する（必要な場合）
rails db:seed

6. 動作確認¶

セットアップが完了したら、アプリケーションを起動して既存の機能が動くことを確認します。

典型的な起動コマンド例:

環境 コマンド例
Node.js npm run dev / npm start
Python / Django python manage.py runserver
Ruby on Rails rails server
Java / Spring Boot ./gradlew bootRun

起動後、ブラウザで http://localhost:3000（ポート番号はプロジェクトによる）を開いて画面が表示されればOKです。

トラブルシューティング（よくあるエラーと対処）¶

エラーの種類 よくある原因 対処法
command not found ツールがインストールされていない 必要なツール（Node.js, Pythonなど）をインストールする
Cannot connect to database DB設定が違う / DBが起動していない .env の設定値を確認、DBサービスを起動する
Module not found npm install などを忘れた 依存パッケージのインストールコマンドを再実行する
ポートがすでに使用されている 別プロセスが同じポートを使っている 起動中のアプリを終了するか、ポートを変更する

チェックリスト（環境セットアップ完了の確認）¶

□ セットアップ手順書を確認した
□ リポジトリを取得（clone / pull）した
□ 依存パッケージをインストールした
□ 環境変数・設定ファイルを配置した（Gitにコミットしていない）
□ アプリケーションを起動できた
□ ブラウザで画面が表示された（または動作を確認できた）

困ったら¶

環境構築で詰まることはよくあります。1人で30分〜1時間考えてもわからなければ、
エラーメッセージをそのままコピーしてチームメンバーに相談してください。
「エラーが出て詰まっています」ではなく、「このエラーが出ています：〇〇」と伝えると解決が早まります。

成果物の例（社員管理システム）¶

以下は「社員管理システム」の開発環境をセットアップした場合の、確認・実施記録の例です。

セットアップ実施記録（記入例）¶

【開発環境セットアップ記録】

担当者  : 山田 太郎
実施日  : 2025-06-02
対象    : 社員管理システム（employee-management）

■ リポジトリ取得
  実行コマンド: git clone https://github.com/your-org/employee-management.git
  結果: ✅ 正常に取得完了

■ 依存パッケージのインストール
  実行コマンド: ./gradlew build
  結果: ✅ BUILD SUCCESSFUL

■ 環境変数・設定ファイルの配置
  テンプレートから .env を作成: cp .env.example .env
  設定した内容:
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=employee_management_dev
    DB_USER=postgres
    DB_PASSWORD=（ローカルのDBパスワード）
    APP_PORT=8080
  結果: ✅ 設定完了（.env はGitにコミットしていないことを確認）

■ データベースのセットアップ
  実行コマンド:
    ./gradlew flywayMigrate
  結果: ✅ マイグレーション完了（テーブル12件作成）

■ アプリケーション起動確認
  実行コマンド: ./gradlew bootRun
  確認URL: http://localhost:8080
  結果: ✅ ログイン画面が表示された

■ 既存機能の動作確認
  確認した機能:
    ✅ ログイン（admin / password123 でログイン成功）
    ✅ 社員一覧画面の表示
    ✅ 部署一覧画面の表示
