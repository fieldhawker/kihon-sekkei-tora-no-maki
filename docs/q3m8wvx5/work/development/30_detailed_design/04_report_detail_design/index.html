帳票詳細設計¶

このドキュメントの目的¶

帳票詳細設計は、基本設計で定義した帳票（PDF・Excel・CSV等の出力物）の出力ロジック・データソース・集計計算・フォーマットを詳細化する作業です。

帳票とは: 印刷・ダウンロードできる書類のことです。例えば「請求書」「納品書」「売上レポート」などがこれに当たります。

作業の流れ¶

1. 帳票一覧の確認
2. 帳票項目とデータソースのマッピング
3. 集計・計算ロジックの定義
4. 改ページ・ソート・出力条件の定義
5. 出力ファイル形式の確認

STEP 1: 帳票一覧の確認¶

基本設計書に記載された帳票の一覧を確認し、詳細設計の対象となる帳票をリストアップします。

帳票ID 帳票名 出力形式 出力契機 担当STEP
RPT-001 請求書 PDF 手動（ボタンクリック） STEP 2〜5
RPT-002 月次売上レポート Excel 自動（月末バッチ） STEP 2〜5
RPT-003 注文データエクスポート CSV 手動 STEP 2〜5

STEP 2: 帳票項目とデータソースのマッピング¶

目的¶

帳票に表示される各項目が「どのテーブル・カラムから取得するか」を定義します。

なぜ必要か: 帳票の項目名と DBのカラム名が必ずしも一致しないためです。例えば帳票に「お客様名」と書いてあっても、DBには customer_name というカラム名で保存されています。この対応を明確にすることで、製造担当者が迷わず実装できます。

記入例：請求書（RPT-001）¶

帳票項目 表示例 データソース（テーブル.カラム） 備考
請求書番号 INV-2024-001 invoices.invoice_number —
発行日 2024年3月15日 invoices.issue_date YYYY年MM月DD日形式で表示
支払期限 2024年4月15日 invoices.due_date YYYY年MM月DD日形式で表示
請求先名 株式会社〇〇 customers.company_name —
請求先住所 東京都〇〇区... customers.address —
商品名 Webシステム開発 invoice_items.item_name 明細行（複数行あり）
単価 ¥500,000 invoice_items.unit_price カンマ区切り・円記号付き
数量 1 invoice_items.quantity —
金額（小計） ¥500,000 計算値（単価 × 数量） STEP 3で定義
消費税 ¥50,000 計算値（小計 × 税率） STEP 3で定義
合計金額 ¥550,000 計算値 STEP 3で定義

データ取得SQL（請求書）¶

-- 請求書ヘッダー情報
SELECT
    i.invoice_number,
    i.issue_date,
    i.due_date,
    c.company_name,
    c.address
FROM
    invoices i
    INNER JOIN customers c ON i.customer_id = c.id
WHERE
    i.id = :invoice_id;

-- 請求書明細情報
SELECT
    item_name,
    unit_price,
    quantity,
    (unit_price * quantity) AS subtotal
FROM
    invoice_items
WHERE
    invoice_id = :invoice_id
ORDER BY
    sort_order ASC;

チェックリスト¶

  • すべての帳票項目に対してデータソースを定義したか
  • 計算値（合計・小計など）の計算ロジックを STEP 3 で定義しているか
  • データが存在しない場合（NULLの場合）の表示を定義したか（空白・ハイフン・「なし」など）

STEP 3: 集計・計算ロジックの定義¶

目的¶

帳票内の集計・計算処理（小計・税計算・合計など）のロジックを定義します。

注意点: 消費税の計算は「切り捨て・切り上げ・四捨五入」のどれを使うかで金額が変わります。必ず業務要件に従って定義してください。

記入例：請求書の金額計算¶

計算項目 計算式 端数処理 備考
明細行の小計 単価 × 数量 なし（整数） invoice_items の各行ごとに計算
請求金額合計（税抜） 全明細行の小計の合計 なし SUM(unit_price × quantity)
消費税額 請求金額合計 × 0.10 切り捨て 小数点以下を切り捨て
請求金額合計（税込） 税抜合計 + 消費税額 なし —

計算の擬似コード¶

1. 明細行リストを取得する

2. 各明細行について:
   小計 = 単価 × 数量

3. 税抜合計 = 全明細行の小計の合計

4. 消費税額 = FLOOR(税抜合計 × 0.10)
   ※ FLOOR = 小数点以下切り捨て

5. 税込合計 = 税抜合計 + 消費税額

チェックリスト¶

  • すべての計算項目の計算式を定義したか
  • 端数処理（切り捨て・切り上げ・四捨五入）を明記したか
  • 消費税率はシステム設定から取得するか、固定値か確認したか

STEP 2: 帳票項目とデータソースのマッピング¶

社員名簿（RPT-EMP-001）¶

帳票項目 表示例 データソース（テーブル.カラム） 備考
出力日 2025年05月25日 システム日付 YYYY年MM月DD日形式
社員番号 E000042 employees.emp_code —
氏名 田中 太郎 employees.last_name + employees.first_name 姓と名の間にスペースを挿入
部署名 人事部 departments.name employee_departments経由でJOIN
雇用区分 正社員 employees.employment_type コード値を名称に変換
入社日 2020年04月01日 employees.hire_date YYYY年MM月DD日形式
勤続年数 5年1ヶ月 employees.hire_date から計算 STEP3で定義
ステータス 在籍中 employees.status コード値を名称に変換

データ取得SQL（社員名簿）¶

SELECT
    e.emp_code,
    e.last_name,
    e.first_name,
    d.name AS department_name,
    e.employment_type,
    e.hire_date,
    e.status
FROM
    employees e
    LEFT JOIN employee_departments ed ON e.id = ed.employee_id
        AND ed.end_date IS NULL
    LEFT JOIN departments d ON ed.department_id = d.id
WHERE
    e.deleted_at IS NULL
    AND (:department_id IS NULL OR ed.department_id = :department_id)
    AND e.status = '在籍中'
ORDER BY
    d.sort_order ASC,
    e.emp_code ASC;

STEP 3: 集計・計算ロジックの定義¶

勤続年数の計算（社員名簿）¶

計算項目 計算式 端数処理 備考
勤続年数（年） （本日 − 入社日）÷ 365.25 の整数部分 切り捨て うるう年を考慮して365.25で除算
勤続月数 （勤続日数 − 勤続年数 × 365.25）÷ 30.44 の整数部分 切り捨て 「X年Xヶ月」の「月」部分 
1. 出力日と入社日の差を日数で算出する（total_days）

2. 勤続年数（年） = FLOOR(total_days / 365.25)

3. 残余日数 = total_days - (勤続年数 × 365.25)

4. 勤続月数 = FLOOR(残余日数 / 30.44)

5. 表示文字列 = "{勤続年数}年{勤続月数}ヶ月"
   ※ 勤続年数が0の場合は「{勤続月数}ヶ月」と表示

部署別在籍者数レポートの集計（RPT-EMP-003）¶

計算項目 計算式 備考
在籍者数合計 各部署の在籍中社員数の合計 COUNT(employees.id) WHERE status = '在籍中'
正社員数 各部署のemployment_type='full_time'の社員数 —
契約社員数 各部署のemployment_type='contract'の社員数 —
