STEP 2: テストケースの作成手順・テンプレート¶

全体フロー: STEP 1 → STEP 2 → STEP 3 → STEP 4 → STEP 5 → STEP 6 → STEP 7

このSTEPの目的¶

テストで確認する内容を事前に一覧化し、抜け漏れを防ぎます。

テストケースとは「何をどうテストして、どういう結果になればOKか」を書いた設計書です。
テストケースがないままテストを進めると、確認漏れや「何をテストしたか」が後から追えなくなります。

テストケースの項目¶

項目 説明 記入例
テストケースID 管理用の番号 TC-001
テスト対象 テストする機能・処理名 ログイン機能
確認観点 何を確認するか 正常系：正しいID/PWでログインできること
前提条件 テスト前に用意すべきデータや状態 テスト用ユーザーがDBに登録済み
テスト手順 操作の手順（番号をつけて具体的に） 1. ログイン画面を開く 2. IDに「testuser」を入力 3. PWに「pass1234」を入力 4. ログインボタンを押す
期待する結果 どうなっていれば合格か ホーム画面に遷移し、「testuser」の名前が表示される
結果（実施後記入） OK / NG （実施後に記入）
備考 特記事項・不具合IDなど （必要に応じて記入）

テストの3つの観点¶

テストケースを作るときは、以下の3つの観点を必ず含めてください。

正常系（せいじょうけい）¶

「正しい使い方をしたときに、期待通りの結果になるか」 を確認するテストです。

ログイン機能の例: 正しいIDとパスワードを入力してログインできることを確認

異常系（いじょうけい）¶

「おかしな操作・入力をしたときに、適切なエラーになるか」 を確認するテストです。

本番では、ユーザーが想定外の操作をすることは珍しくありません。
異常な入力に対してシステムが壊れたり、エラーメッセージなしに止まったりしないよう確認します。

ログイン機能の例: - 間違ったパスワードを入力したとき → エラーメッセージが表示される - IDを空欄のままログインボタンを押す → 「IDを入力してください」と表示される - 存在しないIDを入力したとき → エラーメッセージが表示される

境界値（きょうかいち）¶

「制限の上限・下限ちょうどのときでも正しく動くか」 を確認するテストです。

バグは「ちょうど上限のとき」「ちょうど下限のとき」に潜みやすいです。

パスワード文字数が「8〜20文字」の仕様の場合:

テスト内容 期待する結果
7文字のパスワード エラー（短すぎる）
8文字のパスワード OK（下限ちょうど）
20文字のパスワード OK（上限ちょうど）
21文字のパスワード エラー（長すぎる）

テストケース作成の手順¶

手順1: テスト対象の処理を洗い出す¶

詳細設計書を読んで、テストすべき処理・機能を一覧にします。

例（ログイン機能の場合）: - ID・パスワードの入力と照合 - ログイン成功時の画面遷移 - ログイン失敗時のエラー表示 - 入力値のバリデーション（空欄・文字数など）

手順2: 各処理に対して3観点のテストケースを考える¶

処理ひとつひとつに対して、「正常系・異常系・境界値」それぞれのテストケースを考えます。

処理 正常系 異常系 境界値
ID・PW照合 正しいID・PWでログイン成功 誤ったPWでエラー表示 （この処理に境界値はなし）
PW文字数チェック 8文字以上20文字以下でOK 7文字以下でエラー・21文字以上でエラー 8文字・20文字でOK・7文字・21文字でエラー

手順3: テスト手順を具体的に書く¶

「他の人が読んでも同じ手順でテストできる」レベルで具体的に書きます。

NG例（曖昧すぎる）:

ログイン画面でログイン操作をする

OK例（具体的）:

  1. ブラウザで http://localhost:8080/login を開く
  2. IDフィールドに「testuser」と入力する
  3. パスワードフィールドに「pass1234」と入力する
  4. 「ログイン」ボタンをクリックする

手順4: 期待する結果を具体的に書く¶

「どうなっていればOKか」を明確に書きます。

NG例（曖昧すぎる）:

ログインできる

OK例（具体的）:

ホーム画面（/home）に遷移し、画面右上に「testuser」と表示される

TC-002       |           |         |         |           |             |      |

記入例（ログイン機能）¶

TC-ID テスト対象 確認観点 前提条件 テスト手順 期待する結果
TC-001 ログイン 正常系：正しいID/PWでログインできる testuser / pass1234 がDBに登録済み 1. ログイン画面を開く 2. ID=testuser、PW=pass1234 を入力 3. ログインボタンを押す ホーム画面に遷移し「testuser」が表示される
TC-002 ログイン 異常系：誤ったPWでエラーになる 同上 1. ログイン画面を開く 2. ID=testuser、PW=wrongpass を入力 3. ログインボタンを押す 「IDまたはパスワードが違います」と表示される
TC-003 ログイン 境界値：PW 7文字（下限未満）でエラーになる 同上 1. ログイン画面を開く 2. ID=testuser、PW=pass123（7文字）を入力 3. ログインボタンを押す 「パスワードは8文字以上で入力してください」と表示される

テストケース作成後はレビューしてもらう¶

作成したテストケースは、リーダーや先輩にレビューしてもらってから実施に進むのがベストです。

レビューで確認してもらうこと: - 抜け漏れがないか（確認すべき観点が網羅されているか） - 手順・期待結果が具体的で曖昧さがないか - 前提条件が正しく書かれているか

このSTEPの完了チェック¶

  • テスト対象の処理を詳細設計書から洗い出した
  • 正常系・異常系・境界値のテストケースを作成した
  • テスト手順・期待する結果を具体的に記載した
  • テストケースをリーダーにレビューしてもらった

【成果物例】社員管理システム — テストケース一覧（社員登録機能）¶

テスト対象: 社員登録機能¶

仕様概要: - 社員番号: 必須、数字8桁（例: 00000001） - 氏名: 必須、1〜50文字 - 部署: 必須、マスターから選択 - 入社日: 必須、日付形式（YYYY-MM-DD） - メールアドレス: 必須、メール形式 - 社員番号の重複は不可（一意制約）

テストケース一覧¶

TC-ID テスト対象 確認観点 前提条件 テスト手順 期待する結果 結果 備考
TC-001 社員登録 正常系：全項目を正しく入力して登録できる 社員番号「00000099」が未登録の状態 1. 社員登録画面を開く 2. 社員番号「00000099」を入力 3. 氏名「田中 一郎」を入力 4. 部署「営業部」を選択 5. 入社日「2026-04-01」を入力 6. メールアドレス「tanaka@example.com」を入力 7. 「登録」ボタンをクリック 社員一覧画面に遷移し、「登録が完了しました」と表示される。一覧に田中 一郎が表示される
TC-002 社員登録 異常系：社員番号が空欄のまま登録しようとするとエラーになる なし 1. 社員登録画面を開く 2. 社員番号を空欄のままにする 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「社員番号を入力してください」とエラーメッセージが表示され、登録されない
TC-003 社員登録 異常系：社員番号が既存と重複している場合エラーになる 社員番号「00000001」がDBに登録済み 1. 社員登録画面を開く 2. 社員番号「00000001」を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「この社員番号はすでに登録されています」とエラーメッセージが表示され、登録されない
TC-004 社員登録 異常系：メールアドレスの形式が不正な場合エラーになる なし 1. 社員登録画面を開く 2. メールアドレスに「tanaka_example」（@なし）を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「正しいメールアドレスの形式で入力してください」とエラーメッセージが表示される
TC-005 社員登録 境界値：氏名 1文字（下限ちょうど）で登録できる 未登録の社員番号を使用 1. 社員登録画面を開く 2. 氏名に「山」（1文字）を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 登録が完了し、一覧に「山」が表示される
TC-006 社員登録 境界値：氏名 50文字（上限ちょうど）で登録できる 未登録の社員番号を使用 1. 社員登録画面を開く 2. 氏名に50文字の文字列を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 登録が完了する
TC-007 社員登録 境界値：氏名 51文字（上限超え）でエラーになる なし 1. 社員登録画面を開く 2. 氏名に51文字の文字列を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「氏名は50文字以内で入力してください」とエラーメッセージが表示される
TC-008 社員登録 異常系：社員番号が8桁未満（7桁）の場合エラーになる なし 1. 社員登録画面を開く 2. 社員番号に「0000001」（7桁）を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「社員番号は数字8桁で入力してください」とエラーメッセージが表示される
TC-009 社員登録 異常系：社員番号に数字以外（英字）を入力した場合エラーになる なし 1. 社員登録画面を開く 2. 社員番号に「ABCDEFGH」を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「社員番号は数字8桁で入力してください」とエラーメッセージが表示される
TC-010 社員登録 異常系：入社日の形式が不正な場合エラーになる なし 1. 社員登録画面を開く 2. 入社日に「20260401」（スラッシュなし）を入力 3. 他の必須項目を入力 4. 「登録」ボタンをクリック 「正しい日付形式（YYYY-MM-DD）で入力してください」とエラーメッセージが表示される
