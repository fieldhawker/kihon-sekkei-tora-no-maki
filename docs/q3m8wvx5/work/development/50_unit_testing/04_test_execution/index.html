STEP 4: テストの実施・結果記録¶

全体フロー: STEP 1 → STEP 2 → STEP 3 → STEP 4 → STEP 5 → STEP 6 → STEP 7

このSTEPの目的¶

作成したテストケースを1件ずつ実施し、結果を記録します。

テストは「なんとなく動いた気がする」ではなく、テストケース通りに操作して、結果を記録することが大切です。
記録が残ることで、「テストを実施した証拠」になり、後から不具合が発生したときの調査にも役立ちます。

テスト実施の基本ルール¶

  1. テストケースを上から順番に実施する
    飛ばしたり、順番を変えたりしない。（前の手順が次の手順の前提になっていることがあるため）

  2. 1件ずつ丁寧に実施する
    「2件まとめてやって大丈夫そうだった」という確認の仕方は抜け漏れの原因になります。

  3. 操作はテストケースの手順通りに行う
    手順以外の操作を追加しない。テスト対象が変わってしまうことがあります。

  4. 結果をその場で記録する
    後でまとめて記録しようとすると、詳細を忘れます。実施した直後に記録してください。

結果の記録方法¶

OK（合格）の場合¶

期待した結果と一致した場合は「OK」と記録します。

結果欄 記録内容
OK 期待通りの結果になった

特にコメントがなければOKと記入するだけで問題ありません。

NG（不合格）の場合¶

期待した結果と異なる場合は「NG」と記録し、実際に何が起きたかを詳しく記録します。

記録する内容 説明
実際の結果 実際に何が起きたか（エラーメッセージの内容なども含む）
発生手順 どの手順のときにNGになったか
スクリーンショット NG時の画面を保存する（ファイル名にTC-IDを含める）
ログ エラーログがあれば保存する

NG記録の例:

テストケースID: TC-002
結果: NG
実際の結果: 誤ったパスワードでログインしたが、エラーメッセージが表示されず、
            ホーム画面に遷移してしまった。
発生手順: 手順3でPWを「wrongpass」と入力し、手順4でログインボタンを押した直後
証跡: TC-002_NG_screenshot.png（ホーム画面に遷移した状態の画面）

スクリーンショットの撮り方¶

Windowsの場合¶

方法 操作
画面全体 PrintScreen キー（クリップボードにコピー）
アクティブウィンドウのみ Alt + PrintScreen
範囲を選んでキャプチャ Windows + Shift + S（切り取り＆スケッチ）

保存のコツ: - ファイル名にテストケースIDを含める
例: TC-002_NG_login_error.png - 保存フォルダを決めておく
例: テスト証跡/TC-002/

ログの確認・保存方法¶

エラーが発生したとき、アプリのログにエラーの詳細が記録されています。
ログを確認することで、不具合の原因調査がスムーズになります。

ログの場所はプロジェクトによって異なります。リーダーに確認してください。

よくある場所: - アプリのコンソール画面（ブラウザのデベロッパーツール → Consoleタブ） - サーバーのログファイル（/var/log/app/error.log など） - アプリの管理画面にあるエラーログ

ブラウザのデベロッパーツールの開き方（Chrome）: 1. F12 キーを押す 2. 「Console」タブを開く 3. エラーメッセージを確認する（赤文字のものがエラー）

テスト結果の記録シート（記入例）¶

TC-ID テスト対象 確認観点 実施日 実施者 結果 備考
TC-001 ログイン 正常系：正しいID/PWでログイン 2026-05-25 田中 OK

テスト中によくある疑問¶

Q. テスト手順通りに操作したのに、テストケースでは想定していない挙動をした。どうする？¶

→ それは不具合の可能性があります。「期待した結果と違う」と判断したらNGとして記録してください。
「設計書に書いてないから確認しようがない」と思ったら、リーダーに相談してください。

Q. テストケースの手順が古くて、実際の画面と少し違う。どうする？¶

→ 手順の意図に合わせて実施してOKです。ただし、「手順のここが変わっていた」という記録をメモに残しておき、テストケースを修正してください。

Q. 同じテストケースを何度実施してもNGになる。バグ？環境の問題？¶

→ まずはテストデータやテスト環境が正しいか確認します。それでも再現するなら、バグとして記録して報告します。

このSTEPの完了チェック¶

  • 全テストケースを実施した
  • 全テストケースにOK/NGを記録した
  • NGになったテストケースについて、実際の結果・手順を詳しく記録した
  • NGのスクリーンショット・ログを保存した
  • NGになったテストケースをリーダーに報告した
