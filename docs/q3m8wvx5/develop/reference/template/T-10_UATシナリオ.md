## UATシナリオの読み方

- **シナリオID**: UAT-XXX の形式で採番します
- **前提条件**: シナリオを実施する前に満たしておく必要がある状態
- **操作手順**: 顧客が実際に行う操作を番号順に記載します
- **期待結果**: 操作後に画面・システムに表示されるべき結果
- **結果**: Pass（合格） / Fail（不合格） / Skip（未実施）を記入します
- **指摘内容**: Failの場合に何が問題だったかを記入します

## 詳細シナリオ

### UAT-001: 正常ログインとログアウト

**業務背景**: システムを使い始める前提の基本操作。この操作ができないと業務が一切始まらない。

**前提条件**:
- アカウント（一般ユーザーロール）が事前に作成されている
- ブラウザは【例: Google Chrome 最新版】を使用する

**操作手順**:

| ステップ | 操作 | 期待結果 |
|--------|-----|--------|
| 1 | `https://stg.example.com` にアクセスする | ログイン画面が表示される |
| 2 | メールアドレスとパスワードを入力して「ログイン」ボタンを押す | ホーム画面に遷移し、ログインしたユーザー名が右上に表示される |
| 3 | 右上のユーザー名をクリックし「ログアウト」を選択する | ログイン画面に戻る |
| 4 | ブラウザの「戻る」ボタンを押す | ログイン画面が表示される（認証が必要なページには戻れない） |

**結果**: □Pass  □Fail  □Skip

**指摘内容（Failの場合）**:
```
（何が問題だったかを具体的に記入してください）
例: ステップ2でログインボタンを押しても何も起きない。エラーメッセージも表示されない。
```
