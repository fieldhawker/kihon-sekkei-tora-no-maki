Sample — UI Components

🧩 UIコンポーネント状態一覧

ボタン・入力欄・チェックボックスなど基本部品が持つすべての状態を視覚的に確認できます。設計時に「この状態は考慮したか？」のチェックに使ってください。

💡
すべてのUIには複数の状態がある
初心者が設計書で見落としがちなのが「状態」の設計です。例えばボタン1つでも「通常・ホバー・フォーカス・ローディング・無効」の5状態があります。 設計段階でこれらを定義しておかないと、実装後に「ローディング中にクリックできてしまう」などのバグが発生します。

1 ボタンの状態と種別

🔘

ボタンの状態デモ

通常（Normal）
標準の状態。クリック可能
ホバー（Hover）— マウスを乗せたとき
色が少し暗くなり、クリック可能であることを示す
フォーカス（Focus）— キーボードフォーカス時
Tabキーで移動した際に表示。キーボード操作者に必須
ローディング中（Loading）— 実際にクリックして確認
クリックすると2秒間ローディング状態になります
無効（Disabled）— 操作不可
半透明でクリック不可。「なぜ無効か」をツールチップや説明文で伝える
プライマリボタン
最も重要なアクション。1画面に1つが原則（「保存」「送信」「次へ」など）
セカンダリボタン
補助的なアクション。プライマリの隣に置く（「キャンセル」「戻る」など）
デンジャーボタン（赤）
取り消せない操作（削除・退会など）。クリック前に確認ダイアログを出す
ゴーストボタン（テキスト調）
重要度が低いアクション。リンクの代わりとして使う（「詳細」「もっと見る」など）

サイズは「コンテキスト」で選びます。カード内の小さなアクションは小サイズ、 フォームの送信ボタンは中〜大サイズ。全部同じサイズにしないことが重要です。

⚠️
1画面にプライマリボタンは1つが原則
プライマリボタン（青）が複数あると、ユーザーは「どちらを押すべきか」迷います。 複数ある場合は1つをプライマリに、残りをセカンダリかゴーストに格下げしてください。

2 テキスト入力欄の状態

✏️

6つの状態を設計に反映する

入力欄も複数の状態があります。特に「エラー状態」「読み取り専用」は見落とされがちです。

プレースホルダーは入力例を示す（必須マークや説明はラベルに書く）
青いボーダーとフォーカスリングで「ここに入力中」を示す
入力内容が見えている状態。フォントサイズはプレースホルダーと同じに
⚠️ メールアドレスの形式が正しくありません
グレーアウトで操作不可を示す。なぜ無効かの理由も伝える
内容は見えるがクリックしても変更できない（参照番号など）
💡
disabledとreadonlyの違い： disabledはフォーム送信時に値が含まれません。readonlyは送信時に値が含まれます。 参照番号など「見せたい・送りたい・変えさせたくない」ならreadonly、 「非活性で無視してほしい」ならdisabledを使います。

3 チェックボックス・ラジオボタン・セレクト

チェックボックス（複数選択可）

indeterminate状態は「一部選択（親チェックボックスで子が混在）」のときに使います

ラジオボタン（単一選択）

ラジオは「必ずどれか1つを選ぶ」場合に使います。「どれも選ばない」が許容なら使いません

セレクトボックス
トグルスイッチ（ON/OFF）
通知を受け取る（ON）
ダークモード（OFF）

トグルスイッチは即時反映の設定に使います。「保存ボタンが必要な設定」には使いません

4 状態設計チェックリスト

コンポーネント必要な状態見落としがちな状態
ボタン 通常・ホバー・アクティブ（押下中）・無効 ローディング中（送信中）、フォーカス（Tab移動）
テキスト入力 通常・フォーカス・入力済み・エラー・無効 読み取り専用、パスワード表示/非表示切替
チェックボックス 未選択・選択済み・無効 indeterminate（一部選択）、全選択/全解除
ラジオボタン 未選択・選択済み・無効 デフォルト選択値の設定（何も選ばれていない状態を許容するか）
セレクトボックス 未選択・選択済み・無効 「-- 選択してください --」のプレースホルダー選択肢
リンク 通常・ホバー・訪問済み フォーカス、外部リンク（新規タブ）のアイコン
テーブル行 通常・ホバー 選択済み行、ソート中の列ハイライト、読み込み中（スケルトン）

5 アクセシビリティ — フォーカス状態を消してはいけない

❌
よくある間違い：outline: none; で全消し
CSSで * { outline: none; } や button:focus { outline: none; } を書いてしまう人がいます。 これはキーボードユーザー（視覚障害者・マウスなし操作者）がフォーカス位置を見失う原因になり、 アクセシビリティ違反です。
✅
正しいアプローチ：:focus-visible を使う
button:focus-visible { outline: 3px solid #93c5fd; outline-offset: 2px; } のように :focus-visible を使うと、マウスクリック時はアウトラインが出ず、 キーボード操作時のみアウトラインが表示されます。見た目とアクセシビリティを両立できます。
⌨️

キーボード操作の確認方法

1

Tabキーを押す

フォーカスが次のインタラクティブ要素（ボタン・リンク・入力欄）に移動するか確認

2

フォーカスリングを確認

現在フォーカスされている要素に視覚的なアウトライン（リング）が表示されているか確認

3

Enterキーで操作

ボタンはEnter・Spaceで実行できるか確認。フォームはEnterで送信できるか確認

4

Shift+Tabで逆戻り

逆順にフォーカスが移動するか確認。途中でフォーカスが罠にはまらないか確認
