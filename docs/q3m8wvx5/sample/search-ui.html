🔍 検索UI

検索機能のUIパターン・デバウンス・絞り込み・アクセシビリティを解説するサンプルです。

⚠️ キー入力ごとにAPIを叩くのは厳禁： 10文字入力するたびに10回リクエストが発生します。ユーザー体験も悪化しサーバー負荷も増大します。 デバウンス処理（最後の入力から300ms待機してから1回だけAPIを呼ぶ）が必須です。

⏱️デバウンスとは

デバウンスとは「一定時間、新しいイベントが来なかったら処理を実行する」仕組みです。検索入力に適用することで、APIリクエストを大幅に削減できます。

❌ 悪い例（input毎にAPI呼び出し）

// input イベントで毎回API呼び出し
// "apple" と入力すると5回リクエスト発生
input.addEventListener('input', () => {
  fetch(`/api/search?q=${input.value}`)
    .then(r => r.json())
    .then(render);
});

✅ 良い例（300ms待機後1回だけ）

// デバウンス関数を使う
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const search = debounce((query) => {
  fetch(`/api/search?q=${query}`)
    .then(r => r.json())
    .then(render);
}, 300);

input.addEventListener('input', () => search(input.value));

🔗検索条件のURL反映

💡 URLに検索条件を含めることでブックマーク・シェアが可能になります：
/products?q=ノート&category=電子機器&minPrice=1000&maxPrice=50000&inStock=true
ページ読み込み時にURLパラメータを読み取り、フォームと検索結果を復元することで、URLを共有するだけで同じ検索状態を再現できます。 
// URLに検索条件を反映する
function pushSearchState(params) {
  const url = new URL(window.location.href);
  Object.entries(params).forEach(([k, v]) => {
    if (v) url.searchParams.set(k, v);
    else url.searchParams.delete(k);
  });
  // ページ遷移せずにURLだけ更新（ブラウザ履歴にも追加される）
  history.pushState(null, '', url.toString());
}

// ページ読み込み時にURLから条件を復元する
function restoreSearchState() {
  const params = new URLSearchParams(location.search);
  document.getElementById('filterKeyword').value = params.get('q') || '';
  document.getElementById('filterCategory').value = params.get('category') || '';
  // … 他のフィールドも同様に復元
  if (params.get('q')) applyFilters(); // 条件があれば即検索
}

♿アクセシビリティ

検索UIは特にスクリーンリーダー利用者への配慮が重要です。以下の属性を付与することで、視覚障害のあるユーザーも検索機能を利用できます。

<!-- role="search" で検索領域をランドマークとして宣言 -->
<form role="search" aria-label="商品検索">
  <input
    type="search"
    aria-label="検索キーワード"
    aria-describedby="search-hint"
    aria-controls="search-results"
    autocomplete="off"
    autocorrect="off"
    spellcheck="false"
  >
  <span id="search-hint" class="sr-only">
    キーワードを入力すると自動的に検索します
  </span>
</form>

<!-- aria-live="polite" で結果変更をスクリーンリーダーに通知 -->
<div
  id="search-results"
  role="region"
  aria-label="検索結果"
  aria-live="polite"
  aria-atomic="true"
>
  <!-- 結果件数を含めることでスクリーンリーダーが読み上げる -->
  <p class="sr-only">3件の結果が見つかりました</p>
  <!-- 結果一覧 -->
</div>
✅ アクセシビリティチェックリスト
  • role="search" を検索フォームに付与する
  • 入力フィールドに aria-label または <label> を必ず付ける
  • 結果コンテナに aria-live="polite" と aria-atomic="true" を付ける
  • 結果件数をテキストで明示する（"3件見つかりました"）
  • ローディング中は「検索中…」のテキストをライブリージョンに出力する
  • クリアボタンに aria-label="検索をクリア" を付ける
