非同期処理・ローディング状態

idle / loading / success / error の4状態管理と各UIパターン

⚠️ 設計原則： 非同期処理には必ず 4つの状態 がある。 すべての状態に対応するUIを設計する。
idle（初期）→ loading（処理中）→ success（成功）/ error（失敗）
loading と error だけ設計してしまい、idle（まだ操作していない状態）を忘れるのがよくあるミス。

🗂️4状態の設計原則

💤
Idle（初期）
まだ何も操作していない状態。プレースホルダーや案内文を表示する。ローディングは表示しない。
Loading（処理中）
APIリクエスト中。操作ボタンを無効化してスピナーかスケルトンを表示。連打を防ぐ。
✅
Success（成功）
データを表示する。成功のフィードバック（トースト等）を出す。次のアクションを案内する。
❌
Error（失敗）
エラーの原因と対処法を示す。リトライボタンを置く。ユーザーを行き詰まらせない。

🖱️ボタンのローディング状態デモ

💾データを保存するボタン（成功/失敗がランダム）

クリックするとボタンが無効化されスピナーが表示され、2秒後に成功か失敗かがランダムに決まります。 失敗した場合はリトライできます。

🔒連打防止デモ

1回クリックすると即座に無効化されます。処理が完了するまで何度押しても反応しません。 これにより重複送信を防ぎます。

💀スケルトンスクリーン vs スピナー

使い分け： スケルトンスクリーンはレイアウトが予測できる場合（ユーザー一覧・カードリストなど）に使う。 コンテンツが見えている錯覚を与え、体感速度が速くなる。 スピナーはレイアウトが不定の場合や、処理待ち（保存・削除など）に使う。

↑ レイアウトが決まらない場合はシンプルにスピナー

❤️楽観的UI更新デモ

楽観的UI更新とは： APIの応答を待たずに、成功を前提として即時にUIを更新する手法。 ユーザーがボタンを押した瞬間にカウントが増える。 その後APIが失敗した場合は元の値に戻す（ロールバック）。 いいね・ブックマーク・ゴミ箱移動などに適している。
❤️いいねボタン（楽観的更新）
※ クリック直後にカウントが変わり、70%の確率でAPI成功・30%で失敗(ロールバック)します

✅設計チェックリスト

📝非同期UI 実装前の確認事項
  • idle状態を設計したか？ 「まだデータがない」状態の表示を決めたか
  • loading中はボタンを無効化しているか？ 連打・二重送信を防止しているか
  • loading中のUXを選んだか？ スケルトン vs スピナー vs ボタン内スピナー
  • エラーの種類ごとにメッセージを用意したか？ ネットワークエラー・認証エラー・バリデーションエラーは別メッセージ
  • リトライボタンを置いたか？ エラー時に「もう一度試す」手段をユーザーに与えているか
  • 長時間処理にはキャンセル手段があるか？ ユーザーが途中でやめたい場合の対処
  • 楽観的UI更新のロールバック処理を書いたか？ API失敗時に元の状態に戻すロジック
  • 並列リクエストの競合を防いだか？ 古いレスポンスが新しいレスポンスを上書きしないよう対処（AbortController等）
  • ページング・無限スクロール実装時、追加読み込み中を表示しているか？
  • 成功後のフィードバックを出しているか？ 保存完了・削除完了トーストなど

💻4状態管理の実装パターン

💻React/Vue で使える状態管理パターン
// ❌ よくある間違い: 成功/失敗の boolean だけで管理
const [isLoading, setIsLoading] = useState(false);
const [hasError, setHasError]   = useState(false);
// → idle状態と success状態の区別ができない

// ✅ 正しい: 4状態を1つの変数で管理
type AsyncStatus = 'idle' | 'loading' | 'success' | 'error';
const [status, setStatus] = useState<AsyncStatus>('idle');
const [data,   setData]   = useState(null);
const [error,  setError]  = useState(null);

async function fetchData() {
  setStatus('loading');
  setError(null);
  try {
    const result = await api.getUsers();
    setData(result);
    setStatus('success');
  } catch (err) {
    setError(err.message);
    setStatus('error');
  }
}

// JSX での条件分岐
if (status === 'idle')    return <IdlePlaceholder />;
if (status === 'loading') return <SkeletonScreen />;
if (status === 'error')   return <ErrorMessage msg={error} onRetry={fetchData} />;
return <DataList items={data} />;
