📄 D-03 詳細API仕様書（OpenAPI）— 作成ガイド

🔧
推奨ワークフロー
① openapi.yaml を作成 → ② Swagger UI でブラウザ確認 → ③ Prism でモックサーバー起動 → ④ フロントエンドがモックに対してUI開発 → ⑤ バックエンド実装完了後に本物に切り替え 
openapi: 3.0.3
info:
  title: 経費精算システム API
  version: 1.0.0

paths:
  /api/v1/expenses:
    post:
      summary: 申請書作成
      security:
        - bearerAuth: []
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/ExpenseCreateRequest'
      responses:
        '201':
          description: 作成成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ApiResponse'
        '400':
          description: バリデーションエラー
        '401':
          description: 認証エラー

components:
  schemas:
    ExpenseCreateRequest:
      type: object
      required: [title, amount, date]
      properties:
        title:
          type: string
          maxLength: 100
          description: 申請タイトル
        amount:
          type: integer
          minimum: 1
          description: 金額（円）
        date:
          type: string
          format: date
          description: 使用日（YYYY-MM-DD）
ツール用途コマンド例
Swagger UIOpenAPIの可視化・確認docker run -p 8080:8080 -e SWAGGER_JSON=/openapi.yaml swaggerapi/swagger-ui
Prismモックサーバー起動npx @stoplight/prism-cli mock openapi.yaml
openapi-generatorクライアントコード自動生成openapi-generator-cli generate -i openapi.yaml -g typescript-axios

🔀 D-11 連携シーケンス図（Mermaid）— 作成ガイド

💡
Mermaid記法を使うと、テキストで書いたフローが自動的に図に変換されます。GitリポジトリのMarkdownに埋め込んでコードと一緒にバージョン管理できます。
Mermaid記法（入力）
sequenceDiagram
  actor User as ユーザー
  participant FE as フロントエンド
  participant API as APIサーバー
  participant DB as データベース
  participant Mail as メールサーバー

  User->>FE: 申請書提出ボタン押下
  FE->>API: POST /api/v1/expenses
  API->>API: バリデーション
  API->>DB: INSERT expenses
  DB-->>API: 挿入成功
  API->>Mail: 承認依頼メール送信
  Mail-->>API: 送信完了
  API-->>FE: 201 Created
  FE-->>User: 提出完了メッセージ表示
レンダリング結果
sequenceDiagram actor User as ユーザー participant FE as フロント participant API as APIサーバー participant DB as DB User->>FE: 申請書提出 FE->>API: POST /expenses API->>DB: INSERT DB-->>API: 成功 API-->>FE: 201 Created FE-->>User: 完了表示

🗄️ D-09 マイグレーションスクリプト（Flyway）— 管理方針

ファイル命名規則説明
V1__create_users.sqlV{バージョン}__{説明}.sql（バージョンは昇順）
V2__create_expenses.sqlテーブル作成 DDL
V3__add_index_expenses.sqlインデックス追加
V4__alter_users_add_column.sqlALTER TABLE でカラム追加
🚫
Flywayの鉄則: 一度適用したマイグレーションファイルは絶対に編集しないこと。変更が必要な場合は必ず新しいバージョンのファイルを追加してください。 
-- V4__alter_users_add_column.sql
-- ユーザーテーブルに部署IDカラムを追加
ALTER TABLE users
  ADD COLUMN department_id BIGINT,
  ADD CONSTRAINT fk_users_department
    FOREIGN KEY (department_id) REFERENCES departments(id);

COMMENT ON COLUMN users.department_id IS '所属部署ID';
