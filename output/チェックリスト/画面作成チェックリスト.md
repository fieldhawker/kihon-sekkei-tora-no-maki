# 画面作成チェックリスト

## 要件・UX設計

- [ ] 画面の目的・ユーザーゴールが明確になっている
- [ ] 対象ユーザー（ロール・権限）が定義されている
- [ ] 画面遷移図・フローが設計されている
- [ ] 各項目の入力形式（テキスト・選択・日付など）が定義されている
- [ ] 必須項目・任意項目が明確になっている
- [ ] 初期値・デフォルト値が定義されている

## バリデーション

- [ ] 必須チェックが実装されている
- [ ] 文字数制限が定義・実装されている
- [ ] 入力形式チェック（数値・日付・メールアドレスなど）が実装されている
- [ ] 相関チェック（開始日 ≤ 終了日など）が実装されている
- [ ] サーバーサイドでも同等のバリデーションが実装されている
- [ ] エラーメッセージが具体的でユーザーに分かりやすい

## データ表示・操作

- [ ] 一覧画面の表示件数上限・ページネーションが定義されている
- [ ] ソート・フィルタ条件が定義されている
- [ ] 大量データ時のパフォーマンスが考慮されている
- [ ] 楽観的ロック・排他制御が考慮されている（更新系）
- [ ] 削除時の確認ダイアログが実装されている
- [ ] 保存・送信前の確認フローが定義されている

## セキュリティ

- [ ] 認証・認可チェックが実装されている（未認証アクセスの排除）
- [ ] ロールに応じた表示制御・操作制御が実装されている
- [ ] XSS対策が実施されている（出力のエスケープ）
- [ ] CSRF対策が実施されている（トークン検証）
- [ ] 直接URL入力・パラメータ改ざんによる不正アクセスが防止されている
- [ ] 個人情報・機密情報の表示がマスキングされている

## アクセシビリティ・ユーザビリティ

- [ ] キーボード操作（Tab移動・Enter送信）が適切に動作する
- [ ] エラー時はスクロールしてエラー箇所にフォーカスが当たる
- [ ] ローディング中の状態が視覚的に分かる（スピナーなど）
- [ ] 処理成功時のフィードバック（完了メッセージなど）が実装されている
- [ ] 戻るボタン・パンくずリストなど、ナビゲーションが整備されている
- [ ] レスポンシブ対応の要否が定義されている

## 非機能・運用

- [ ] ページタイトル・メタ情報が適切に設定されている
- [ ] ブラウザ対応範囲が定義・確認されている
- [ ] 画面表示の応答時間の目標値が定義されている
- [ ] エラーページ（404・500など）のデザインが用意されている
- [ ] 操作ログ（誰が・いつ・何を）が記録される設計になっている

## テスト

- [ ] 正常系の入力・送信・遷移が動作確認されている
- [ ] 各バリデーションエラーの表示が確認されている
- [ ] 権限外ユーザーのアクセスが拒否されることが確認されている
- [ ] 複数タブ・並行操作時の挙動が確認されている
- [ ] 実機・対象ブラウザでの表示確認が実施されている
