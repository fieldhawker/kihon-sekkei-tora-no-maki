# テスト実施手順書

> **このフォルダの目的**: 実装完了後のテスト実施フェーズを、経験の浅いSEが迷わず進められるようにまとめた手順書です。「テストケースの実行方法」だけでなく「バグの記録・修正サイクル・完了判定」まで含みます。

---

## テストフェーズの全体像（V字モデル）

設計フェーズとテストフェーズは対応しています。

```
要件定義          ←──────────────────→  受入テスト（UAT）
  基本設計        ←────────────────→  システムテスト
    詳細設計      ←──────────────→  結合テスト
      実装（コーディング） ←──→  単体テスト
```

各テストが「どの設計フェーズで決めた仕様を検証するか」を意識することで、何をどこまでテストすればよいかの判断基準が明確になります。

---

## テストレベルの定義と担当

| テストレベル | 目的 | 主担当 | 参照設計書 |
|-----------|-----|------|---------|
| **単体テスト** | 関数・クラス単体が仕様どおりか | プログラマー | D-01 詳細機能設計書 |
| **結合テスト** | API・画面・DB がつながって動くか | SE・プログラマー | D-03 API仕様書、D-06 物理ER図 |
| **システムテスト** | 業務シナリオ全体が要件を満たすか | SE | 基本設計書（B-01〜B-19）、要件定義書 |
| **受入テスト（UAT）** | 顧客・エンドユーザーが承認できるか | 顧客・テスト担当 | 要件定義書 |
| **リリース・カットオーバー** | 本番環境への安全な切替と稼働確認 | SE・インフラ担当 | カットオーバー手順書（T-11）、運用設計書（B-16）|

## 全体の流れ

```
実装完了
    │
    ▼
┌─────────────────────────────────────┐
│  STEP 1: 単体テスト実施              │
│  関数・クラス単位での検証             │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│  STEP 2: 結合テスト実施              │
│  API〜DB・API〜画面の連携確認         │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│  STEP 3: システムテスト実施           │
│  業務シナリオ全体の通し確認           │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│  STEP 4: テスト完了判定              │
│  残存バグの評価・リリース判定         │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│  STEP 5: 受入テスト（UAT）           │
│  顧客による業務シナリオ確認・承認     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│  STEP 6: リリース・カットオーバー     │
│  本番環境への切替・スモークテスト     │
└──────────────────┬──────────────────┘
                   │
                   ▼
        運用フェーズへ移行
```

| STEP 2 | 結合テスト実施 | 5〜8日 | SE・プログラマー |
| STEP 3 | システムテスト実施 | 3〜5日 | SE |
| STEP 4 | テスト完了判定 | 1日 | SE・PL |
| STEP 5 | 受入テスト（UAT） | 5〜10日 | 顧客担当者・SE |
| STEP 6 | リリース・カットオーバー | 1〜2日（準備含む） | SE・インフラ担当・PL |
