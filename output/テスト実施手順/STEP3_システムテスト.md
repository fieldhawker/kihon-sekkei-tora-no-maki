# STEP 3: システムテスト実施

**このステップの目的**: 業務シナリオ全体を通して、システムが要件定義書・基本設計書の仕様を満たしているかを確認します。

---

## システムテストの特徴

単体テスト・結合テストとの違いを理解してから実施してください。

| テストレベル | 視点 | 確認単位 |
|-----------|-----|--------|
| 単体テスト | プログラマー視点 | 関数・クラス |
| 結合テスト | SE視点 | API・DB・画面の繋がり |
| **システムテスト** | **ユーザー視点** | **業務シナリオ全体** |

---

## テストシナリオの作成と実行

### テストシナリオとは

「実際のユーザーが行う一連の操作」を端から端まで通した確認です。

**例: 商品購入シナリオ**
1. ログインする
2. 商品を検索する
3. 商品をカートに追加する
4. 購入手続きをする
5. 確認メールが届いていることを確認する
6. 管理画面で注文が登録されていることを確認する

### テストシナリオの書き方

```
シナリオID: ST-001
機能カテゴリ: 購入フロー
前提条件: テストユーザー（test@example.com）でログイン済み
          テスト商品（商品ID:101）が在庫あり状態であること

手順:
  1. トップページから「商品一覧」を開く
     期待結果: 商品一覧が表示される
  2. 商品名「テスト商品A」を検索する
     期待結果: 商品ID:101 が一覧に表示される
  3. 「カートに追加」ボタンをクリックする
     期待結果: 「カートに追加しました」のメッセージが表示される
  4. カートページを開く
     期待結果: 商品ID:101 が数量1で表示される
  5. 「購入する」ボタンをクリックして購入を確定する
     期待結果: 「ご注文ありがとうございます」の画面が表示される
              注文IDが表示される
  6. test@example.com の受信ボックスを確認する
     期待結果: 注文確認メールが届いている

後処理: テスト注文データを削除する
```

## 非機能テスト（性能テスト）

### 性能テストの実施要否の判断

| 条件 | 判断 |
|-----|-----|
| 同時接続ユーザー数の要件がある（例: 同時50名以上） | 実施必須 |
| ピーク時の処理量が大きい（例: 1日10万件のトランザクション） | 実施必須 |
| レスポンスタイムのSLAがある（例: 95%tile 3秒以内） | 実施必須 |
| 社内のみ・少人数利用 | 省略可 |

### 性能テストツール

| ツール | 特徴 | 向いているケース |
|------|------|-------------|
| k6 | JavaScriptで書ける・CI統合が容易 | API性能テストの第一候補 |
| Apache JMeter | GUI操作・レポート機能が豊富 | 複雑なシナリオ・Javaチームに馴染みがある場合 |
| Locust | Pythonで書ける | Pythonチームの場合 |

### k6 の基本的な使い方

```javascript
// simple-load-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 50,        // 仮想ユーザー数
  duration: '60s', // テスト実行時間
};

export default function () {
  const res = http.get('https://your-app.example.com/api/v1/products');
  check(res, {
    'ステータスコードが200': (r) => r.status === 200,
    'レスポンスタイムが3秒以内': (r) => r.timings.duration < 3000,
  });
  sleep(1);
}
```

```bash
k6 run simple-load-test.js
```

### 性能テスト結果の評価

| 指標 | 合格基準（非機能要件設計書 B-13 の数値に従う） |
|-----|----------------------------------------|
| レスポンスタイム（95パーセンタイル） | 設計書に記載した値以内 |
| エラー率 | 0.1%以下 |
| スループット | 設計した同時接続数でのRPS（Requests Per Second）が要件を満たすこと |

## システムテスト チェックリスト

### 機能テスト

- [ ] 全テストシナリオを実施した
- [ ] 全シナリオでPass（または承認済みの不具合として記録済み）

- [ ] 性能テストのレスポンスタイムが非機能要件（B-13）を満たしている
- [ ] セキュリティ設計書（D-12）の対策が実装されていることを確認した
- [ ] ステージング環境（本番相当）でテストを実施した
