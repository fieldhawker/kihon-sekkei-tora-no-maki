# STEP 2: 結合テスト実施

**このステップの目的**: API・データベース・外部システム・画面が「つながって」正しく動作するかを確認します。

---

## 結合テストの範囲

```
フロントエンド（画面）
        ↕ ← ここを確認
バックエンド（API）
        ↕ ← ここを確認
データベース
        ↕ ← ここを確認（外部連携がある場合）
外部システム
```

---

## APIテストの手順

### ツール: Postman または curl

#### Postmanを使う場合

1. **コレクションを作成する**: プロジェクト用のコレクションを作成し、APIエンドポイントをグループごとに整理する
2. **環境変数を設定する**: `base_url`・`auth_token` 等を環境変数で管理する（`localhost:8080` や `staging.example.com` を切り替えられるようにする）
3. **テストスクリプトを追加する**: 各リクエストの「Tests」タブにレスポンス検証を追加する

```javascript
// Postman テストスクリプトの例
pm.test("ステータスコードが200であること", () => {
    pm.response.to.have.status(200);
});

pm.test("レスポンスにuserIdが含まれること", () => {
    const json = pm.response.json();
    pm.expect(json.userId).to.be.a('number');
});

pm.test("レスポンスタイムが1秒以内であること", () => {
    pm.expect(pm.response.responseTime).to.be.below(1000);
});
```

4. **コレクションランナーで一括実行する**: Run Collection → すべてのAPIを自動実行してPass/Failを確認する

#### curlを使う場合（シンプルな確認）

```bash
# GETリクエスト（認証なし）
curl -X GET http://localhost:8080/api/v1/users/1 \
  -H "Content-Type: application/json"

# POSTリクエスト（JWTトークンつき）
curl -X POST http://localhost:8080/api/v1/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"name": "テストユーザー", "email": "test@example.com"}'
```

## テストデータの管理

### テストデータの準備

結合テスト用のデータは以下の方針で管理します。

| データの種類 | 管理方法 |
|-----------|---------|
| 初期マスタデータ | Flywayのシードスクリプト（`V9999__seed_test_data.sql`）で投入 |
| テストケースごとのデータ | テスト実行前にAPIで作成 または SQLで直接投入 |
| テスト後のクリーンアップ | テスト終了後にテーブルをTRUNCATEまたは特定データを削除 |

> **本番データをテストに使わない**: 個人情報・実際の取引データをテスト環境に持ち込まないでください。テストデータは必ずダミーデータを使います。

### テストデータ設計の方針

- **正常系データ**: 通常の業務を想定したデータ（代表的なユーザー・商品等）
- **境界値データ**: 最大文字数・最小値・空値・null を含むデータ
- **エラーを引き起こすデータ**: 重複するメールアドレス・存在しないIDなど

## 画面×API の結合確認

APIが通った後、実際に画面から操作して確認します。

### 確認手順

1. ブラウザでアプリを開く（ローカルまたはステージング環境）
2. テストケース一覧（D-13）に記載された操作手順を実施する
3. 画面の表示内容・エラーメッセージ・画面遷移が設計書（D-02）と一致することを確認する
4. ブラウザの開発者ツール（F12）でAPIのリクエスト/レスポンスを確認する

### よくある結合テストの失敗パターン

| 症状 | 考えられる原因 | 確認場所 |
|-----|------------|--------|
| 画面にデータが表示されない | APIのURLが間違っている / CORSエラー | ブラウザの開発者ツール > コンソール |
| 「認証エラー」が出続ける | JWTトークンの有効期限・形式の問題 | APIのレスポンス・バックエンドのログ |
| 保存後にデータが更新されない | APIは200を返しているがDBに保存されていない | DBを直接クエリして確認 |
| 日本語が文字化けする | 文字コードの設定が UTF-8 になっていない | DBの文字コード設定・APIヘッダーの charset |
