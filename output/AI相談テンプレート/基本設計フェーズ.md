## テンプレート1: システムアーキテクチャ設計

```
## 相談内容: システムアーキテクチャ設計

私はWebシステムの基本設計を担当するSEです。
システムアーキテクチャの方針を決めようとしています。

### システム概要
- システム名/目的: 【例: 社内経費精算Webシステム】
- 想定ユーザー数: 【例: 同時接続50名、月間ユーザー300名】
- 主な機能: 【例: 申請・承認ワークフロー、ファイルアップロード、外部会計システム連携】

### 制約・前提条件
- 開発チーム規模とスキル: 【例: SE3名。Javaは得意だがGoは未経験】
- 予算感: 【例: インフラ月額10万円以内】
- リリース期限: 【例: 6ヶ月後】
- インフラ方針: 【例: AWSを使う予定。オンプレは不可】
- 既存システムとの関係: 【例: 既存の会計システム（オンプレ）とCSV連携が必要】

### 特に相談したいこと
【例: フロントエンドをSPA（React）にするかSSR（Next.js）にするか迷っている】
【例: DBはRDBとNoSQLのどちらが向いているか判断してほしい】

上記の条件に合ったアーキテクチャ構成の候補を提示し、それぞれのメリット・デメリットと推奨案を教えてください。
```

## 相談内容: 技術スタック選定

### システム概要
- 規模: 【例: 画面20画面・API40本・テーブル30本程度】

### 制約
- 採用禁止技術: 【例: Railsは社内サポートなしのため採用禁止】

### 特に迷っている選択
【例: バックエンドをSpring BootにするかQuarkusにするか】
【例: ORMをJPA/HibernateにするかMyBatisにするか】

チームスキルと制約を踏まえた技術スタックの推奨案を教えてください。選定理由と、あわせて検討すべきリスクも教えてください。

## テンプレート4: セキュリティ設計レビュー

```
## 相談内容: セキュリティ設計のレビュー

以下のシステムのセキュリティ設計をOWASP Top 10の観点でレビューしてください。

### システム概要
- 公開範囲: 【例: インターネット公開 / 社内ネットワークのみ】
- 扱う個人情報: 【例: 氏名・メールアドレス・購入履歴】
- 認証方式: 【例: JWT（RS256・有効期限15分）+ リフレッシュトークン（30日）】
- 通信方式: 【例: HTTPS（TLS 1.2以上）】

### 設計済みの対策
- SQLインジェクション対策: 【例: JPA（プリペアドステートメント）を使用。SQL文字列連結は禁止】
- XSS対策: 【例: Vue.jsのテンプレートを使用（自動エスケープ）。v-htmlは禁止】
- CSRF対策: 【例: SPA + JWT認証のためCSRFトークン不要と判断】
- 認証: 【例: パスワードはbcryptでハッシュ化（コスト係数12）】
- アクセス制御: 【例: Spring SecurityのROLE_ADMINとROLE_USERで制御】

### 特に確認したいこと
【例: CSRF対策の判断（不要と判断した根拠）が適切かを確認してほしい】

OWASP Top 10の各項目について、上記の設計で対策が十分かをレビューし、不足があれば改善提案をしてください。
```
