## テンプレート3: データベース設計（インデックス・クエリ最適化）

```
## 相談内容: インデックス設計とクエリ最適化

以下のクエリに対して、適切なインデックス設計と最適化のアドバイスをください。

### テーブル構造
orders テーブル:
- id (BIGINT, PK)
- user_id (BIGINT)
- status (VARCHAR: 'pending'/'paid'/'cancelled')
- ordered_at (TIMESTAMP)
- total_amount (DECIMAL)

order_items テーブル:
- id (BIGINT, PK)
- order_id (BIGINT, FK -> orders.id)
- product_id (BIGINT)
- quantity (INT)

### 主要なクエリパターン
1. 特定ユーザーの注文一覧を新しい順に取得（ページネーション）
   SELECT * FROM orders WHERE user_id = ? ORDER BY ordered_at DESC LIMIT 20 OFFSET ?

2. 管理画面での注文一覧（ステータス・日付範囲でフィルタ）
   SELECT * FROM orders WHERE status = ? AND ordered_at BETWEEN ? AND ? ORDER BY ordered_at DESC

3. 注文ID を指定して注文と全明細を取得
   SELECT o.*, oi.* FROM orders o JOIN order_items oi ON o.id = oi.order_id WHERE o.id = ?

### データ規模
- ordersテーブル: 現在10万件。3年後に100万件を想定
- order_itemsテーブル: orders の平均3件の明細

各クエリに対して推奨するインデックス設計と、将来のデータ増加を考慮した注意点を教えてください。
```
