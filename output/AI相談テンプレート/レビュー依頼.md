```
## テンプレート3: SQL・DB設計のレビュー依頼

```
## 依頼: SQL・DB設計のレビュー

以下のSQLまたはDB設計をレビューしてください。

### DB環境
【例: PostgreSQL 15 / データ量: ordersテーブル 約50万件】

### SQL・テーブル定義

```sql
-- ここにSQLを貼り付ける
SELECT
    o.id,
    o.ordered_at,
    u.name AS user_name,
    COUNT(oi.id) AS item_count,
    SUM(oi.quantity * oi.unit_price) AS total
FROM orders o
INNER JOIN users u ON o.user_id = u.id
INNER JOIN order_items oi ON o.id = oi.order_id
WHERE o.status = 'paid'
  AND o.ordered_at >= '2024-01-01'
GROUP BY o.id, o.ordered_at, u.name
ORDER BY o.ordered_at DESC
LIMIT 50 OFFSET 0;
```

### 確認したいこと
【例: 上記クエリのパフォーマンスに問題がないか】
【例: 適切なインデックスを教えてほしい】

以下の観点でレビューしてください:
1. クエリのパフォーマンス問題の有無
2. 推奨するインデックス設計
3. データが増加した場合の注意点
