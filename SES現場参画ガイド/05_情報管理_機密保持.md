# 情報管理・機密保持

> **このファイルの目的**: 客先の情報を適切に扱い、NDA（秘密保持契約）を守るための実践的なガイドラインをまとめます。情報漏洩は民事・刑事上の責任につながるため、特に注意が必要です。

---

## SESエンジニアが触れる機密情報の種類

現場で扱う情報は多岐にわたります。どの情報が「機密」かを意識してください。

| カテゴリ | 具体例 |
|--------|------|
| **顧客・ユーザー情報** | 氏名・住所・メールアドレス・電話番号・購買履歴 |
| **システム・技術情報** | ソースコード・設計書・インフラ構成・APIキー・パスワード |
| **ビジネス情報** | 売上データ・事業計画・未発表のサービス・取引先情報 |
| **組織・人事情報** | 社員の評価・給与・組織の内部課題 |
| **セキュリティ情報** | 脆弱性情報・認証情報・セキュリティ設定 |

> **「公開されていなければ機密と思え」** が基本の考え方です。「これは機密か？」と迷ったら機密として扱ってください。

## 情報の取り扱いルール

### デジタルデータ

| 行為 | 原則 |
|-----|-----|
| 個人所有のPCへのコピー | **原則禁止**（客先・自社のポリシーを確認する） |
| USBメモリへのコピー | **原則禁止**（現場のセキュリティポリシーを確認する） |
| クラウドストレージ（個人Google Drive等）へのアップロード | **禁止** |
| 業務メールの個人メールへの転送 | **禁止** |
| スクリーンショットの個人端末への保存 | 原則禁止（許可を取る） |
| 設計書・仕様書の印刷 | 現場のルールに従う。持ち帰りは原則禁止 |

### 物理的な資料

- 印刷した資料は現場外に持ち出さない
- 不要になった印刷物はシュレッダーにかける
- ホワイトボードの内容を写真に撮る場合は許可を確認する

### 口頭での情報取り扱い

- 現場内での会話は外に漏らさない（電車内・カフェでの業務会話は特に注意）
- SNS・ブログへの現場情報の投稿は行わない（たとえ匿名でも）
- 家族・友人への業務詳細の話は避ける

## 開発ツール・AIツールの利用に関する注意

### ChatGPT・Claude等のAIへの入力

AIへ業務上の情報を入力することは、情報が外部のサーバーに送信される行為です。

| 入力しても問題ない | 入力してはいけない |
|---------------|---------------|
| 一般的な技術的質問（コードの書き方等） | 実在する顧客名・システム名 |
| ダミーデータを使った相談 | 実際のソースコード（機密性のあるもの） |
| パブリックな技術情報の確認 | 個人情報・顧客データ |
| 汎用的な設計パターンの相談 | APIキー・認証情報 |
|  | 未公開の事業計画・製品情報 |

> **現場のAI利用ポリシーを最初に確認する。** 「AIツールの使用禁止」「業務情報の入力禁止」など、明示的なポリシーがある現場も増えています。

### GitHubへのコード公開

- 現場のコード・設計書をpublicリポジトリにプッシュしない
- 個人のGitHubに業務コードをバックアップする行為も原則禁止

## 自社・家庭での注意事項

### リモートワーク時

- 自宅での作業でも機密保持義務は変わらない
- 家族が画面を見えてしまう環境での作業に注意する
- VPNの接続を忘れずに行う
- 業務端末を家族が使用しないようにする

### 自社への報告時

- 自社担当に状況報告する際も、客先の機密情報を必要以上に開示しない
- 自社の資料・提案書への客先情報の転記は最小限にとどめる

---

## 参画終了時・退場時の対応

退場時は、以下を確実に実施してください。情報の持ち出しが後でトラブルになるケースがあります。

### データの扱い

- [ ] 業務用に使用した自社・個人PCから客先データを完全に削除した
- [ ] クラウドに同期した業務ファイルがあれば削除した
- [ ] メールの添付ファイル・ダウンロードしたファイルを削除した
- [ ] 開発環境・仮想マシンに客先データが残っていないか確認した

### アクセス権の返却

- [ ] 客先システム・リポジトリ・サーバーのアカウントを無効化依頼した
- [ ] VPNの接続設定を削除した（または現場の指示に従った）
- [ ] 入館証・IDカード・キーを返却した

### 「学習用として残したい」という誘惑への注意

「勉強になったコードを手元に残したい」「設計書のフォーマットが参考になるので保存したい」という気持ちは理解できますが、**これも機密情報の持ち出しに該当します**。

- コードや設計書は客先の知的財産
- 退場後に「あの現場の設計書を参考に新しい現場で使った」が漏れると問題になる
- 学習の記録として残したい場合は、**内容の概要や学んだ考え方をメモする程度**にとどめる

## 機密情報漏洩が発生した場合

万が一、情報漏洩が疑われる事態（誤送信・端末紛失・不正アクセス等）が発生した場合は、**隠さず・速やかに報告**することが重要です。

1. 発生した事実を確認・記録する
2. **自社担当に即報告する**（客先への報告は自社が調整する）
3. 拡大を防ぐための初動対応（アカウントの無効化・端末の隔離等）
4. 原因と影響範囲の調査に協力する

> **隠蔽は絶対にしない。** 後から発覚した場合の方がはるかに問題が大きくなります。「報告すると叱られる」と思っても、速やかに報告することが自分の身を守ることにもつながります。
