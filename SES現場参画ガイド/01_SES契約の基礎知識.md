# SES契約の基礎知識

> **このファイルの目的**: SES契約の法的な性質・他の契約形態との違い・自分の権利と義務を理解することで、現場で不当な扱いを受けずに働けるようにします。

---

## 契約形態の比較

IT現場で使われる主な契約形態は3種類です。自分がどれに該当するかを確認してください。

| 契約形態 | 法的区分 | 指揮命令 | 仕事の完成責任 | 労働時間管理 |
|--------|--------|--------|------------|-----------|
| **SES（準委任契約）** | 民法656条 準委任 | **自社（SES会社）** | なし（善管注意義務） | 自社が管理 |
| **派遣（労働者派遣）** | 労働者派遣法 | **客先** | なし | 客先が管理 |
| **請負（受託開発）** | 民法632条 請負 | 自社 | **あり（成果物の完成）** | 自社が管理 |

> **SESと派遣の実務上の違い**: 指揮命令権が「自社にある（SES）」か「客先にある（派遣）」かが法律上の大きな違いです。しかし現場では**SES契約なのに客先が直接業務指示を出す**という「偽装請負」状態になりやすく、グレーゾーンになることがあります。

---

## SES（準委任契約）の重要ポイント

### 善管注意義務とは

準委任契約では「仕事を完成させる義務（請負）」はありませんが、**「その職業・専門家として通常期待される注意を払って業務を遂行する義務（善管注意義務）」** はあります。

- 完成しなかったことで損害賠償を問われることは通常ない
- ただし「明らかに手を抜いた」「必要な確認をしなかった」は問題になりうる
- 「設計書を作ったが品質が低すぎる」場合は義務違反として問われる可能性がある

### 指揮命令は自社から受ける

SES契約では、**作業内容の詳細指示（何を・どの手順で・どのツールで）は自社から受けるのが本来の形**です。

- 客先は「業務の目標・成果の方向性」を示すことができる
- 客先が「○○の実装方法はこうしろ」と細かく指示するのは**偽装請負の疑いがある**
- 実際には多くの現場でグレーゾーンになっている

> **実務的な対処**: 明らかに指揮命令の問題が発生している場合は自社の営業担当に相談する。一般的な技術的アドバイス・レビューの指摘は業務指示とは区別される。

## 契約書で確認すべき項目

現場参画前に、自社と交わした契約書・参画条件書を確認してください。

### 業務範囲

- [ ] 担当する業務の内容が具体的に記載されているか
- [ ] 「その他付随する業務」という曖昧な文言が広すぎないか確認する
- [ ] 担当外の業務（例: インフラ構築、マネジメント業務）が含まれていないか

### 稼働条件

- [ ] 月の稼働時間の上限・下限（精算幅）が明記されているか
  > 例: 「140〜180時間」→ この範囲内なら定額。超過分は別途精算
- [ ] 残業・休日出勤の扱いが明確か
- [ ] 現場の就業時間・場所（常駐かリモートか）が明記されているか

### 単価と精算

- [ ] 月額単価が明記されているか
- [ ] 稼働時間が精算幅を超過・下回った場合の計算方法が明確か
- [ ] 交通費・経費の扱いが明記されているか

### 守秘義務

- [ ] 客先の機密情報の取り扱いについて記載されているか
- [ ] 退場後の守秘義務期間が明記されているか

### 契約終了

- [ ] 契約期間（いつからいつまで）が明記されているか
- [ ] 契約終了（退場）の通知期限が明記されているか（例: 1ヶ月前通知）
- [ ] 自己都合での退場申し出の手続きが分かるか

## 稼働時間の精算幅（よくある仕組み）

SES契約では「月160時間を超えた分は追加費用を請求できる」「140時間を下回ると単価が減額される」という精算幅が設けられることが多いです。

```
【例: 精算幅 140〜180時間、月額単価 60万円の場合】

稼働 170時間 → 精算幅内 → 60万円（変わらず）
稼働 190時間 → 超過10時間 → 60万円 + (60万÷160 × 10時間) = 追加あり
稼働 130時間 → 下回り10時間 → 60万円 − (60万÷160 × 10時間) = 減額
```

> **注意**: 「精算幅の下限を下回りそうなとき」でも、**意味のない残業をして時間を埋めるのは問題**です。稼働時間が足りない場合は自社担当に相談し、正直に申告することが重要です。

## よくある「グレーゾーン」と対処

| 状況 | 法的に言うと | 実務的な対処 |
|-----|-----------|-----------|
| 客先担当者が直接「この実装はこうしろ」と細かく指示してくる | 偽装請負の疑いがある | まずは従いつつ、自社担当に状況を報告する |
| 「SES契約だが残業は客先の指示で強制される」 | 時間管理は自社が行うべき | 自社担当を通じて客先と調整してもらう |
| 契約書に書いていない業務を依頼される | 契約外作業になる可能性 | `04_スコープ管理.md` を参照 |
| 客先から「次の契約更新はしない」と直接言われた | 退場通知は通常自社経由で行う | 自社担当に即報告し、対応を自社に任せる |

## 契約終了（退場）に関する知識

### 退場を申し出る場合

1. **自社担当に相談する**（客先に直接言わない）
2. 契約書に記載の通知期限（通常1〜3ヶ月前）を守る
3. 引き継ぎ資料を丁寧に作成する（次に入るエンジニアへの配慮）
4. 退場日まで通常どおり業務を遂行する

### 客先から契約終了を通告された場合

- 通常は自社経由で通知が来る
- 急な退場要求（1週間以内など）は契約書に違反する可能性がある → 自社担当に確認
- 退場理由が「能力不足」の場合、具体的な指摘内容を確認し、自社担当を交えて話し合う

### 退場時にやること

- [ ] 担当業務の引き継ぎ資料を作成する
- [ ] 客先から借りたPC・資料・入館証等を返却する
- [ ] 自分が作成・使用したデータのうち客先に残すべきものを整理する
- [ ] 自分の私物を持ち帰る（私物と業務データを混在させない）
- [ ] 退場後は守秘義務を継続して守る

## SES法律用語の簡易解説

| 用語 | 意味 |
|-----|-----|
| **準委任契約** | 仕事の完成ではなく「作業の遂行」を約束する契約。SES契約の法的根拠 |
| **善管注意義務** | 専門家として通常期待される注意を払う義務 |
| **偽装請負** | 形式上は請負・SESだが実態として客先の指揮命令を受けている違法状態 |
| **指揮命令権** | 業務の具体的な指示を出す権限。SESでは自社にある |
| **精算幅（上下限時間）** | 月の稼働時間の許容範囲。この範囲内は単価固定 |
| **二重派遣** | SES会社が別の会社に転売する行為。労働者派遣法で禁止 |
| **常駐** | 客先のオフィスで勤務すること |
| **常駐なし（リモート）** | 自社や自宅から客先プロジェクトに参加すること |
